# 夜間窓口｜SAFARI観測記録局

自分の観測記録に誤配された大量のダイアログを、マイクへの息で吹き飛ばす3〜5分のブラウザ謎解きゲーム。
静的HTML/CSS/JavaScriptのみで動作する。

## 結論

コアは「助けを求めるための音声伝言」だと思わせ、実際には呼気で画面上のダイアログそのものを吹き飛ばす反転体験。

初版の紙資料・朱印中心の意匠、解法を直接示す通知文、結果に影響しない2分タイマー、返還印ボタンは削除した。

## 4点セット

| 最初に見せる違和感 | プレイヤーの誤解 | 誤解を解く体験 | 最後の回収 |
|---|---|---|---|
| 自分の観測記録なのに、すべて「夜間管理人宛」のダイアログで覆われている | 管理人を呼び出せば誤配を処理してもらえる | 夜間窓口へ移動し、話し声では動かない受付票が呼気で浮いて飛ぶ | 元の記録へ戻り、同じ呼気で全ダイアログを除去。下の記録が呼気を「生存確認」と判定し、プレイヤーを記録の外へ返す |

## 世界観の核

**SAFARI観測記録局は、生き物の呼気だけを本人の生存証明として扱い、証明された被観測者を記録の外へ返す。**

- ギミックの必然: 呼気は単なる入力ではなく、観測対象が生きて画面のこちら側にいる証明。
- 行動の必然: 管理人への音声伝言を試す行為が、意図せず本人確認になる。
- 遠回りの必然: 空の夜間窓口へ行ったことで、一枚の受付票を安全に試し、元画面の異常へ戻る手がかりを得る。
- 数値の物語化: 420msの持続判定は瞬間音ではなく一呼吸を確認する時間。複数回の呼気は覆われた記録を段階的に露出させる。
- 意図した例外: 観測記録そのものは透過光台に固定されているため飛ばず、誤配されたダイアログだけが画面外へ剥がれる。

ゲーム内ではこの設定を説明せず、最後に「生存確認：呼気あり」とだけ表示する。

## 体験フロー

1. 透過光式記録台に、自分の観測記録と8件の管理人宛ダイアログが表示される。
2. ダイアログの×を押しても、宛先本人の処理待ちとして再表示される。
3. 「夜間管理人を呼ぶ」から別画面の夜間窓口へ移動する。
4. マイク許可と0.7秒の環境音測定後、押して話せる音声伝言を試す。
5. 普通に話すと事務的な不達応答。息っぽい音では受付票が膨らみ、窓口の外へ飛ぶ。
6. 「窓口は、最初から空でした」と分かり、観測記録へ戻る。
7. 同じ呼気を使って8件のダイアログを3回に分けて吹き飛ばす。
8. 下から現れた記録が「生存確認：呼気あり」「記録外へ返還」に変わり、クリア。

## ヒントと救済

- 初期画面: 解法を示す文言なし。×を押した場合のみ再表示という失敗反応を返す。
- 夜間窓口1回失敗: 「伝言を受理しました。管理人の確認は明朝です。」
- 夜間窓口2回失敗: 「伝言は届いています。受付票の端だけが震えています。」
- 失敗後45秒: 「窓口が拾ったのは、言葉よりも空気に近い音でした。」
- 観測記録へ戻った後2回失敗: 「夜間窓口で、動いたものを思い出してください。」
- 同一画面で4回失敗またはマイク拒否・非対応: 手動整理を提案。
- 手動整理: ダイアログを約0.48秒長押しして左右へ56px払う。キーボードではフォーカス後Delete/Backspace。

代替操作は失敗前に先出ししない。

## 息判定とマイクの扱い

Web Audio APIで音声を送信せず、端末内で特徴量だけを解析する。

- 音声窓口への初回接続時に約700msの環境音を測定。
- 音量: `max(環境音 + 8dB, -42dBFS)` より大きい。
- スペクトル平坦度、高域比率、ゼロ交差率の3条件中2つ以上。
- 上記を420ms以上維持。
- クリッピング率2%以上を除外。

初版と異なり、マイクストリームは長押しのたびに取得し、指を離した時点で全トラックを停止する。待機中にブラウザのマイク使用状態を残さない。

## ビジュアル方針

- 主題: 夜間のSAFARI保存修復室にある透過光式記録台。
- 単一の仕事: 誤配ダイアログの下から自分の観測記録を取り戻す。
- 色:
  - 夜間室 `#10191b`
  - 修復台 `#26383b`
  - 透過光 `#cce6df`
  - ダイアログ膜 `#e9efeb`
  - 風圧 `#f06b52`
  - 生存確認 `#69c6a6`
- 書体: 観測記録は明朝、操作は可読性の高いゴシック、回線情報は等幅。
- 署名的表現: 息候補の間だけダイアログが薄膜のように膨らみ、成功時に風圧等高線が下から走って画面外へ剥離する。
- 紙ノイズ、方眼紙、朱印、常時揺れなどコアの物理反応と競合する装飾は使わない。

## ローカル起動

マイクはHTTPSまたはlocalhostでのみ利用できる。

    cd apps/akashic-games
    python3 -m http.server 8000

`http://localhost:8000/notification-gale/` を開く。

## デバッグ

`?debug=1` を付けると特徴量HUDと決定的な検証APIを有効化する。

- `T`: 夜間窓口の受付票を呼気成功扱いにする
- `B`: 観測記録のダイアログを呼気成功扱いにする
- `V`: 現在画面の音声不達を記録する
- `M`: 手動整理を解放する
- `window.gameDebug.tutorialBreath()`
- `window.gameDebug.blow()`
- `window.gameDebug.fail()`
- `window.gameDebug.fallback()`
- `window.gameDebug.getState()`

## アクセシビリティ

- マイク操作はポインター長押しとSpace/Enterに対応。
- 手動整理は長押しスワイプとDelete/Backspaceに対応。
- マイク状態と結果は色だけでなく文言でも通知。
- 覆われた記録はダイアログ除去まで`aria-hidden`にする。
- 応答は`aria-live`で読み上げる。
- `prefers-reduced-motion`を尊重する。
