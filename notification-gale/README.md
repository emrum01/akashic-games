# 未来からの同意要求｜SAFARI FUTURE RELAY

未来のあなたが、いまのあなたの脳に処置を施し「飲酒を強制的にやめさせよう」としている。処置には
今日中に本人の同意が必要だが、その同意書は押し寄せる胡散臭いWeb広告に塗りつぶされている。管理人に
頼んでも助けてもらえず、プレイヤーは放置される。だが冒頭で開いた「マイク回線」だけが生きていて――
端末に強く息を吹きかけると、広告が物理的に吹き飛ぶ。それに自力で気づき、現れた同意書に「承認する」。
3〜5分のブラウザ謎解き。静的HTML/CSS/JavaScriptのみで動作する。見た目は「2010年頃の古めかしいWeb」に
「未来からの通信」が異物として届く対比。世界観の鍵は「未来はWebを物理的に操作できる」。

## 結論（謎の骨子）

**「吹き飛ばす」操作をUIで教えない。** マイクボタンも長押しの説明もない。プレイヤーは
「管理人に頼めば消してもらえる」と思い込むが、管理人は助けてくれない（確認は9日以降）。
その後は何のボタンも出ず、広告に阻まれたまま放置される。環境に散らした遠回しの示唆
（ヘッダーのローマ字 `FUKITOBASU`／「回線は大きな風速に弱い」というステータス／紛れたヒント受信）
から、**自分の息（＝未来からの物理干渉）で広告を吹き飛ばす**ことに気づく反転体験。最後は自分の意思で
同意書に署名してクリアする。

## 世界観

**未来はWebを物理的に操作できる。** 未来のあなたは、過去のあなた（プレイヤー）の脳へ処置を施し
飲酒をやめさせようとするが、処置には本人の同意が要る。回線には敵対的広告が紛れ込み同意書を覆う。
広告は×では消えず、管理人は助けない。だが端末の「マイク回線」だけは生きていて、そこから広告に
物理干渉できる――それが未来からの介入経路。マイク回線は「風」を通す＝強い呼気に反応する。

ゲーム内では設定を説明口調で語らず、断片（同意書の文面・管理人の自動応答・環境の示唆・承認の演出）
で伝える。

### 設計意図: デザインの時代差は世界律（恣意パラメータの物語化）

「広告＝現代風／本体UI＝2010年頃の古めかしさ」という見た目の時代差は、手抜きではなく世界設定である。

- **広告が現代風な理由**: 詐欺グループは各時代のWebデザインを熟知し、その時代（＝現代）の見た目で
  マーケしてくる。だから広告は今のネットで見る詐欺・記事広告そっくりで、いやな親近感がある。
- **本体UIが古い理由**: 未来の医師は、記憶を頼りに過去のWebを再現してこの同意画面を組んでいる。
  だから時代がズレて古めかしい＝「未来人による不完全な再現」。
- この差は nazo-design の「恣意パラメータの物語化」に沿った設計。プレイ中は説明せず見せ方で感じさせ、
  クリア時の未来の医師からの一文（「私の記憶にある昔のWebを真似て作った。不格好で申し訳ない」）で
  だけ、そっと回収する（世界律モチーフのクリア回収）。

### 設計意図: 操作を「謎」にする（UIで教えない）

「吹き飛ばす」をボタンやチュートリアルで教えると謎ではなくなる。そこで操作の告知を廃し、代わりに
環境へ遠回しの手掛かりを散らす。地の文・広告面に操作や答えを直書きしない方針（nazo-hint-design）を
守りつつ、詰まったときだけ段階的に後出しする。

### 設計意図: 広告の背景設定（未来技術を過去に売る商売）

**ゲーム内では一切語らない。広告を組むための前提としてのみ用いる。**

未来のマーケティング業者が、時間通信を使って**過去にいる顧客へ未来の製品を売りつける**ビジネスが
存在する。未来技術の商品（10秒で20kg減、若返り、記憶の削除、寿命+15年、脳OS更新、故人との通話など）は
過去の顧客には夢のようだが、購入すると**未来の自分に法外な料金が請求される**。この商売は**違法**だが、
業者は「同一人物が承諾している」ことを盾にし、さらに時代を跨ぐ管轄の問題もあって、**実効的には
なかなか取り締まれない**状況にある。ゲーム内の警告がこれを「**敵対的広告群**」と呼ぶのはこのため
（明確に違法と断罪しきれない、厄介な相手というニュアンス）。

- 広告の**見た目は現代の胡散臭いWeb広告のまま**（記事広告・偽警告・当選ポップアップ・Cookieバナー等の
  現実的フォーマット）。**中身（商品・コピー）だけを未来技術の誇大広告**にして、現代広告の文法で未来の
  製品を売る。
- **請求が未来の自分に行く**匂わせ（「後払い（30年後）」「課金は30年後」「返済は将来のあなた」等の小さな
  注記）を数枚に控えめに仕込み、背景をそれとなく滲ませる（説明はしない）。
- 写真広告の焼き込み帯（赤丸・煽り文）は汎用的なクリックベイトのままでも未来技術コピーと両立する
  （ビフォーアフター＝若返り／体重削減 等は特に相性が良い）。ゲームが描画するコピー（`ad.b`）を未来技術に
  差し替えることで整合させている。

## 体験フロー（すべて同一画面。別画面への遷移はしない）

1. タイトル: 未来のあなたからの通信。「脳に処置を施し飲酒をやめさせる／今日中に本人の同意が必要」＋
   「※広告が紛れ込む場合があります」の免責→「同意書をひらく」。
2. **マイク許可ゲート(micgate)**: 開始直後に2010年風のシステムダイアログ「このページは、未来との
   マイク回線を使用します」を表示。ここで「マイク回線を開く」をタップ＝**実際の `getUserMedia` 許可**を
   取る（iOS のユーザージェスチャ要件もここで満たす）。**何に使うかは明かさない（伏線）。** 許可されれば
   環境音を約0.7秒測定し、以降マイクは**常時オン**になる。拒否・非対応でも先へ進む（後述のフォールバック）。
3. ダウンロード(download): 「未来からの同意書をダウンロードしています」＋同意書本文をターミナル風に
   1文字ずつタイピング（点滅カーソル）。ステージのタップで早送り可（ただし次の警告までで止まる）。
4. **緊急警告(alert)**: タイピングの途中で**画面中央のモーダル警告**が割り込む（旧OS風クリティカル
   アラート＝赤いタイトルバー＋⚠＋太字見出し「敵対的広告群を検知しました」＋本文「これより、大量の
   広告が表示されます」）。**最低表示時間が経つまで閉じられず、早送り連打でもスキップされない**
   （「広告が来る」緊張の立ち上がりを必ず伝える）。「広告を確認する」で閉じると氾濫へ。reduced-motion
   でも必ず表示（点滅のみ抑制）。
5. 氾濫(flood): 敵対的広告が1個ずつ画面を埋め尽くし、同意書を覆い隠す。**その中に1個だけ、クリックできる
   「未署名の受信」（ヒント）が紛れている**（他の広告は×でも無反応）。
6. 検知(detected): 「同意書が広告に埋もれた。× でも消えない。管理人に連絡してみて」。― このとき
   ステータスに「回線は現在、大きな風速に弱い状態です」という**風＝息の遠回しな示唆**が出る。
   マイクが生きていれば、この時点から広告は**強い呼気で吹き飛ばせる**（常時リスニング＋リアルタイム物理）。
7. 連絡(contacted): 管理AIの自動応答は**絶望的**――「同意の代行はできません。確認は最短で9日以降です。
   なお当回線は現在、大きな風速に弱い状態です」。**閉じるとその後は何のボタンも出ない。** 広告に阻まれた
   まま放置され、プレイヤーは自力で「息で吹き飛ばす」に気づくしかない。
8. **吹き飛ばし（気づきと実行）**: マイクは常時オン。端末に**強く息を吹きかける**と、入力レベルに比例した
   放射状の力が働き、広告が中心から外へ物理的に押し出される。弱い息・話し声・定常的な環境音では動かない
   （相対閾値＋定期再キャリブレーション）。息を止めると減衰して少し戻る／その場に残る。
9. **承認（最終アクション）**: 広告が晴れて同意書が現れる。内容を確認し「承認する（同意する）」を押す→
   同意書が「署名済」になりクリア。

## 環境に隠したヒント（遠回し・段階後出し）

操作を直書きしない代わりに、以下を環境へ散らす。

- **ヘッダーの `FUKITOBASU`**: ステータスバーのブランド下に `build FUKITOBASU`（＝吹き飛ばす）を小さく
  目立たない体裁で紛れ込ませる。
- **回線ステータスの一文**: 検知フェーズのガイダンスと管理人の自動応答に「回線は大きな風速に弱い状態」
  （風＝息）を載せる。
- **紛れたヒント受信（任意・2段階の後出し）**: 氾濫の中の1個だけクリックでき、開くと遠回しの示唆が読める。
  第1「未来はWebを物理的に操作できる。この回線は、そのための細い通り道のようだ」→第2「回線は“風”を通す。
  強い空気の流れに、めっぽう弱いらしい」。**操作の直書きはしない。** 読まなくてもクリア可能。
- **常時インジケータ**: マイク使用中は小さなチップで「マイク回線 使用中／音声は録音・送信されません」と
  レベルメーターを表示（操作要素ではない＝プライバシー配慮の告知）。

## 救済（詰み回避）

- **長時間停滞のみ段階ヒント**: 氾濫到達から**約90秒**進展がなければ「大きな風――強い空気の流れに弱い」、
  さらに**約150秒**で「端末に、強く息を吹きかけてみてください」と後出しする。**一度でも吹き飛ばしに成功
  （first_blow）したら以降は出さない。**
- **マイク拒否・非対応**: 冒頭ゲートで許可されなかった場合も先へ進む。管理人に連絡した直後に、手動で払う
  導線（長押し＋外へスワイプ／Delete）を静かに提示する。手動でも広告を払い切れば同意書が現れ、最後は
  「承認する」でクリア。
- 進行保証: 連絡ボタン（record-console, 最前面）は常に押せる。ヒント受信は氾濫の中でも常に可視・タップ可能
  （縦600px/vvh-tiny 含む。物理・手動払いの対象外＝z-index を全広告より上に）。

## マイクの扱いと呼気判定

Web Audio APIで音声を送信せず、端末内で特徴量だけを解析する。実機で確実に拾うため感度優先。

- `getUserMedia` は**冒頭のマイク許可ゲートのタップで取得**（ユーザージェスチャ）。`echoCancellation` /
  `noiseSuppression` / `autoGainControl` を**すべて false**（呼気ノイズが前処理で抑圧されるのを防ぐ）。
- 取得直後に約700msの環境音を測定し、線形RMSのパーセンタイルをベースラインにする。以降マイクは**常時オン**。
- **常時リスニングの誤発火対策**: 「強い息」でないフレームのRMSを蓄積し、約2.5秒ごとに下位パーセンタイルへ
  ベースラインを緩く追従させる（周囲の騒音が変化しても、定常音では動かず「強い息」だけを拾い続ける）。
- **強い息のときだけ広告を動かす**。RMSがベースラインの `strong` 倍（既定3.0）＋ノイズ様（平坦度 か
  ゼロ交差率が高い）を `shold` ミリ秒（既定120ms）維持したときだけ物理ループが広告を押す。弱い息・話し声・
  定常環境音では動かさない。届いているが弱いときはインジケータ／広告のかすかな微振動でフィードバックする。
- 物理: 力 = 強い息のレベル × 放射方向、速度に減衰、押していないときは基準へ弱く戻す
  （`requestAnimationFrame`, transform で移動）。画面外まで押されたら除去。
- バックグラウンド（visibilitychange・pagehide）で全トラック停止。復帰時は取り直す（取得できなければ手動
  フォールバックへ）。
- fake audio 検証で「ボタンなしの強入力でのみ広告が変位／定常ノイズでは不動」を bounding box で確認済み。

## ビジュアル方針

frontend-design スキルを踏まえ、サイバーパンクは廃止。**本体UI（タイトル/同意書/管理人/システム）を
「2010年頃の古めかしいWebサイト」**の質感に、**広告を「現実の胡散臭いWeb広告」の再現**にして、
「日常のうざい広告」と「未来からの通信」を対比させる。

- 本体UI（2010年風）: 灰色の地に白いパネル、青い上罫＋明朝の見出し、青い下線リンク、銀ベベルの
  ボタン（緑の「進む/DL」風CTA）、旧管理画面風の見出しバー、システムフォント（Meiryo/MS PGothic系）。
- 未来からの通信（同意書・注意書き・ヒント受信・管理人）: ネオンではなく**冷たく無機質なスレート**。
- 緊急警告モーダル: 旧OS風クリティカルアラート（赤タイトルバー＋⚠＋太字見出し＋OKボタン）。
- 広告（現実のWeb広告の再現）: 記事風カード／偽OS当選ポップアップ／偽ウイルス警告／Cookieバナー／
  緑のDLボタン／黄色い煽りバナー、様式7種×サイズ3段を混在させ重ねる（全24枚）。
- 広告写真: **codex CLI の image_gen（gpt-image-2）で生成**した写真に ImageMagick で赤丸・赤矢印・
  黄帯煽り・PR表記・JPEG劣化を合成した `ads/adphoto-*.jpg`（全9種）。生成物の出所は `ads/CREDITS.md`。
- 署名後の同意書に「署名済」スタンプ。

## ローカル起動

マイクはHTTPSまたはlocalhostでのみ利用できる。

    cd apps/akashic-games
    python3 -m http.server 8000

`http://localhost:8000/notification-gale/` を開く。

### キャッシュバスター（更新時に手で上げる）

`index.html` は `styles.css` / `game.js` / `analytics.js` を `?v=<版>`（例 `?v=20260712a`）付きで読み込む。
ブラウザが古いJS/CSSをキャッシュして更新が反映されない事故を防ぐため、**これらを変更したら
`index.html` 内の3か所の `?v=` を同じ新しい値へ手で更新する**（日付＋連番の運用でよい。例
`20260712a`→`20260712b`）。3か所を揃えること。

## デバッグ

`?debug=1` で特徴量HUD（rms・ベースライン比・レベル・各閾値）と検証APIを有効化。

判定パラメータは URL クエリで上書きできる（例: `?debug=1&strong=2.4&shold=100`）。
`strong`（押し出し発動の環境音比＝既定3.0）/ `shold`（強い息の必要持続ms＝既定120）/ `ratio` / `flat` /
`zcr` / `clip` / `span`。

Playwright での検証は Chromium 起動フラグ
`--use-fake-device-for-media-stream --use-fake-ui-for-media-stream`
（＋`--use-file-for-fake-audio-capture=<wav>`）で疑似音声を流して行う。強入力の検証時は
`window.gameDebug.setBaseline(0.004)` で環境音基準を固定すると決定的に「強い息」を再現できる。

- `F`: 警告を飛ばして氾濫へ（検証用） / `C`: 管理人に連絡 / `B`: 全広告へ外向き初速 / `M`: 手動払いを解放
- `window.gameDebug.hint()` — ヒント受信を開く / `window.gameDebug.consent()` — 同意書に署名してクリア
- `window.gameDebug.classify(features)` / `setBaseline(rms)` / `getState()`（phase / remaining / micReady /
  micDenied / firstBlowFired 等）

## 広告タップ → 未来サービス詳細（読み物）

広告カードを**短くタップ（クリック）**すると、その広告に対応する「未来のサービス詳細」オーバーレイが
開く（`ADS[].d`）。2010年代の悪質LP × 未来SFの誇大な説明文＋関連画像＋「支払いは30年後の自分」を
表現する料金対比（今のあなた ¥0／30年後のあなた 法外な額＋利息・自動更新・解約不能の注記＋合計請求額の
カウントアップ）で、世界律「未来からの介入は必ず対価を取る」を読ませる。操作ヒントや答えは書かない。

- **操作の共存**: クリック判定は「移動量<10px かつ 350ms 未満の短いタップ」のみ。長押し＋スワイプの手動
  払い・ドラッグ・呼気物理とは衝突しない。× ボタンは閉じる専用（詳細は開かない）。
- **閉じ方**: 右上の × ／ カード外側（バックドロップ）タップ ／ Escape で確実に閉じる。開いている間も
  背景の物理・進行は動き続ける。
- 氾濫に紛れた「未署名の受信」（ヒント）は広告ではないため詳細を持たない。

## アクセシビリティ

- 冒頭のマイク許可はタップで取得。吹き飛ばしは息（マイク）だが、拒否・非対応時は手動払い（長押しスワイプと
  Delete/Backspace）へ静かに誘導する。
- マイク状態と結果は色だけでなく文言でも通知。応答は `aria-live` で読み上げる。
- `prefers-reduced-motion` を尊重する（点滅・アニメーションを抑制。緊急警告モーダルは表示は維持）。
