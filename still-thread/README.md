# 流れないスレ（still-thread）

スクロールしても画面に貼りついて動かない黒い「シミ」。速く流すほど投稿は滲んで流れ、シミだけが鮮明に残る——動かないものこそが本物だと気づく。流れに逆らって**留まる**と、シミは墨字「ながれるな」として完成する。最後、「次スレへ ▶」（流れて先へ行く誘い＝偽物）ではなく、**留まった文字に触れる**とクリア。3〜5分の単発Web謎。

- 公開想定 URL: https://games.escape-safari.com/still-thread/
- 企画の正: [`DESIGN.md`](./DESIGN.md)
- 技術: 単体HTML＋DOM/CSS（Three.js不使用・外部CDN不使用・ビルド不要・静的配信）

## 世界観の核（ゲーム内では説明しない・ここにのみ全文記録）

> このスレッドという怪異においては、流れるもの（スクロールで過ぎ去る投稿・次スレへの誘い）はすべて偽物で、流れに逆らって留まるもの（画面に貼りついたシミ＝文字）だけが本物である。

プレイヤーが最初から無意識にやっていた「スクロール（流す）」という行為そのものが怪異の狙いだった、と反転する。核はプレイヤーに言い当てさせない。体感で一貫していればよい（各画面1文以内の含みに留める）。

## コアギミック

1. **固定層（シミ）**: 上部70%セーフゾーン（32%線）に `position:fixed` で墨のシミ＝**仮名の片割れ（左クリップ）**を置く。スクロールに追従せず常に鮮明・非ブラー。初見は「汚れ・表示バグ」に見える。
2. **補完墨（流れる側）**: スクロールする投稿群の中に**もう片割れ（右クリップ）**が仕込まれ、特定スクロール位置で片割れが1文字に重なる。
3. **速度連動**: スクロール速度に比例して投稿へ `filter: blur()`＋脱色、背景に下方向の水流。**シミは一切ブラーしない**→流すほどシミだけ際立つ。
4. **練習領域「いる」**: 許容誤差広め・ドリフト無し。止まるだけで自動2秒完成（成功体験・伏線。操作説明はしない）。
5. **本番領域「ながれるな」**: 4〜5片が同時に重なる唯一の地点。**下方向ドリフト**が視界を引きずる。位置を合わせて**画面を押さえ続ける（press-and-hold）2秒**でドリフトに逆らい完成。指を離す／スクロールが動くとリセット。
6. **最終選択**: 囮「次スレへ ▶」（脈動・押すとソフト失敗）と、留まって完成した文字（タップ可能な本物）を上部70%内に**分離配置**。留まった文字に触れるとクリア。

## 段階的ヒント（先出し禁止・詰み検知後だけ）

- 練習帯を高速通過し続ける → 「速く流すほど、あの点だけが、はっきりする。」
- 約40秒どのシミも完成しない → 「点は動かない。動いているのは、こっちだ。」
- 本番で位置合わせ＋保持ができない（22秒）→ 「止まって、待ってみたら？ 流されないように。」
- さらに詰まる（40秒）→ 代替操作「長押しで、その場に留まれる。」（失敗後にだけ後出し）
- 「次スレへ」を2回以上押す → 「次へ進むほど、遠ざかる。」

失敗フィードバック: 次スレへ押下=「……流してしまった。」／保持中に流された=「途中で、また流れてしまった。」

## Xアプリ内ブラウザ対応（セーフゾーン）

`threejs-single-file-game` スキルの「セーフゾーン実装」に準拠（DOM版に翻案）:

- `100vh` 不使用。`window.visualViewport.height` を `--app-height` に反映。
- `#fixed-layer` を `visualViewport.offsetTop` 分だけ `translateY` 補正し、URLバー伸縮時もシミが上部70%に留まる。
- 進行必須（シミ・補完墨・完成文字・留まる対象・先人の固定投稿）は上部70%内。下部は囮の「次スレへ」のみ。最終2択は囮と本物を上下に分離し互いを覆わない。
- `env(safe-area-inset-bottom)` を下部余白に算入。最小ビューポート縦600px（390×600・852×600）で成立を確認済み。

## GA4計測イベント

`window.AkashicAnalytics.track(event, params)` で送信（本番ホストのみ、`?debug=1` で console 出力）:

| イベント | 発火 |
|----------|------|
| `game_start` | 「スレをひらく」 |
| `fragment_align` | 練習「いる」完成 |
| `stay_proven` | 本番「ながれるな」保持完成 |
| `decoy_tapped` | 「次スレへ」押下（ソフト失敗、`count` 付き） |
| `game_clear` | 留まった文字に触れてクリア |

## デバッグ・検証

- `?debug=1` で左下にHUD（scrollTop・速度・各シミ完成状態・フェーズ・保持タイマー）を表示。
- `window.__GAME_STATE__` … フェーズ・完成状態・各種実測値を常時公開（debug無しでも）。
- 検証フック（常時利用可）:
  - `window.__DEBUG__.setScroll(y)` … スクロール位置を設定
  - `window.__DEBUG__.forceAlign()` … アクティブな補完墨をシミへ重ねる位置までスクロール
  - `window.__DEBUG__.forceHold()` … 現フェーズの保持完成を即成立
  - `window.__DEBUG__.press(bool)` / `tapStay()` / `tapDecoy()` / `enterMain()` / `enterChoice()`

## アクセシビリティ

- 文字コントラスト確保、完成語・回収文は完成後は常に鮮明（演出だけに依存しない状態表示）。
- `prefers-reduced-motion`: ブラー/ドリフト/水流/脈動を軽減。
- タップターゲット44px以上（CTA・囮・シミのタップ領域）。
- 状態は `aria-live` の status（ヒント・完成）で通知。

## 既知の制約・調整余地

- **本番ドリフトの強さ**（`DRIFT`）は仮置き。テストプレイで詰める。
- `position:fixed` × Xアプリ内ブラウザの実機挙動が最大リスク。ヘッドレスの縮小ビューポートは通過したが、実機（縦600px含む）でシミが上部70%に留まりタップ可能かを最終確認する。
- 画像アセットは後工程。現状はCSSフォールバックで成立。
