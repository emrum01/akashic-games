# 流れないスレ（still-thread）

縦に流れる匿名掲示板の中で、画面に留まり続ける異物だけが本物になる短編Webホラー。

プレイヤーは投稿を読むうち、スクロールに従わない黒いシミへ気づく。スレに引かれて流されかけたあと、自分の操作で留まることで怪異から抜ける。答えを住人の説明や選択肢で教えず、**発見、抵抗、沈黙**をプレイヤー自身の体験として成立させる。

- 公開想定URL: https://games.escape-safari.com/still-thread/
- 体験時間: 数分
- 実装: 単体HTML + DOM/CSS/JavaScript（外部CDN・ビルド不要）
- 仕様の正: [DESIGN.md](./DESIGN.md)

## ファイル

| パス | 役割 |
|---|---|
| `index.html` | ゲーム本体。CSSとJavaScriptを内包 |
| `analytics.js` | GA4計測ラッパー |
| `assets/` | OGPや任意の質感素材。ゲーム進行は素材なしでも成立させる |
| `verify/` | 画面・進行の検証記録 |
| `DESIGN.md` | 現行の演出・体験仕様（SSOT） |

## 変更時に守ること

- 平常な掲示板から始め、異常を先に宣伝しない。
- 怪異を住人に解説させない。プレイヤーの発見を奪うレスを足さない。
- 怖さの中心は「固定された異物」と「スクロールを奪われる感覚」に置く。赤い画面、激しいグリッチ、文字化け、効果音の重ね掛けで補わない。
- 強制移動は短くし、プレイヤーが抵抗して止められるようにする。
- 最後は選択モーダルを出さない。留まって何もしない時間そのものを正解にする。
- Good endでは、怪異の正体・住民が誘う理由・取り込まれずに済んだ理由を結末として開示する。Bad endだけを豪華にしない。
- 全編を意図的な無音で成立させる。音響機能を進行条件にしない。
- `prefers-reduced-motion` では演出を高速再生せず、移動をカットまたは短いフェードへ置換する。

## 検証

ローカルサーバーで開き、実機相当の縦長viewportを中心に確認する。

```bash
python3 -m http.server 8781 --directory apps/akashic-games/still-thread
```

最低限、次を通す。

1. 初見でシミが文字やホラー装飾に見えすぎない。
2. 新着が現在のviewport内で読め、画面外だけで演出が完結しない。
3. 暴走スクロール（pull）へ抵抗でき、暴走中も住人のパニック投稿が読める。
4. 誘導フェーズで「次スレへ」を押さず20秒留まるとGood endへ到達する。
5. 「次スレへ」を押すとBad end（発狂して死ぬ次スレ）へ入り、冒頭の横幅崩れがない。
6. `prefers-reduced-motion` で長距離または高速の強制移動が発生しない。
7. キーボード、タッチ、マウスで進行でき、タップ対象は44px以上ある。
8. `pageerror` がなく、GA4イベントが重複しない。
