# ブックカフェ思想改造ゲーム — 改造後バリエーション設計書

対象: `bookcafe-3d.html` / 実装者: three.js エンジニア
目的: **改造成功後の店内が「ただNPCのセリフが変わるだけ」問題**を解消する。
平積み棚に特定ジャンルの本3冊を入れた瞬間、店内の**客層・店員・空気・オブジェクト**が"化ける"。その変化の面白さを、見た目と振る舞いのバリエーションで作り込む。

---

## 0. 実装の前提（この設計書の読み方）

既存コードから読み取った"部品"の実態。ここに書いた仕様は全部この範囲で作れる。

### 0-1. 座標系・寸法
- 部屋: `ROOM = {minX:-11, maxX:11, minZ:-8, maxZ:8, wallH:4}`。床は y=0、天井 y=4。
- **中央平積み棚**（改造トリガー）: 原点 `(0,0,0)`。天板 y=0.75、スロット3つが `x=-1,0,+1` / `y=0.86`。
- レジカウンター: `(-7.5, y, 5.6)`。店員が居ない側の"奥"は back 壁 `z=-8` 側。
- 通常テーブル&椅子: `[-4,-4] [4,-4] [-4,2] [7,2] [0,-6]`（各 group 原点、椅子は ±1.4 のx）。
- 装飾本棚: 右壁 `x≈10.4`, `z=-4/0/4`（LOS遮蔽物でもある）。
- **プレイヤー**: `(0,0,5.5)` スタート（=手前中央）。

### 0-2. キャラは「プリミティブ人型」（`buildHumanoid`）
現状キャラは Kenney GLTF ではなく `buildHumanoid(opts)` で Box/Cylinder/Sphere から組む人型。`opts = {skin, cloth, cap, apron}`。関節は group.userData に `armL/armR/legL/legR`（各 pivot Group）として露出済み。歩行は `animateHumanoid(h, dt, moving, tempo)` が腕脚を sin で振るだけ。

> **重要な実装方針**: 課題文が挙げる Kenney クリップ名（idle/sit/emote-yes/static 等）は「表現したい振る舞いの意図」として使う。現行リグに GLTF を差すなら**そのクリップ名をそのまま割当**。差さない（プリミティブ継続）なら、下記の**ポーズプリセット**を `armL/armR/legL/legR/head` の `rotation.x/z` で表現する。両対応できるよう、各振る舞いに「クリップ名」と「プリミティブでの近似ポーズ」を併記する。

推奨: `buildHumanoid` に `pose` を追加できるよう、次のヘルパを1つ足す（実装者向け提案）:
```
// ポーズプリセット。名前→各pivotのrotation目標。animateHumanoidの前に適用。
const POSES = {
  static:      { armL:0, armR:0, head:0 },
  sit:         { legL:-1.4, legR:-1.4, /* かつ y を椅子高に */ },
  emoteYes:    { head:'nod', armR:-2.4 /* 腕を上げて力説 */ },
  emoteNo:     { head:'shakeSlow' },
  holdBoth:    { armL:-1.2, armR:-1.2 /* 前に何か捧げ持つ */ },
  leanWhisper: { head:0.5 /* 隣に顔寄せ */ , torsoLean:true },
  pickUp:      { armR:-1.9, torsoBendZ:0.4 },
};
```
"nod/shake" は `update()` 内で `head.rotation.x = Math.sin(t*speed)*amp` の小刻み。既存の `animateHumanoid` と共存させるため、**改造後のNPCは moving=false 固定**（座り/立ち）にして、pose 側で頭・腕を動かす。

### 0-3. 空気（色調）の既存フック
`triggerEvent()` が改造時に呼ぶのは:
- `applyTint(gd.tint)` … 背景・フォグ色・hemiライト・内装マテリアルを tint 側へ lerp。
- `updateMenu(menuTitle, menu)` … メニュー差し替え。
- `showBanner` / `addLog` / `showHint`。

**本設計書の追加分はすべて `triggerEvent()` の末尾から呼ぶ `applyGenreScene(activeGenre)` 1関数に集約する**ことを推奨。この関数が「客の入替 / 店員の変化 / プロップ追加 / 空気の追い演出 / ギャグ」を担う。既存NPC（`npcs[]` / `buildStaticNPCs`）は改造時に**作り替える or 見た目パーツを足す**。

### 0-4. 共通の"化ける"演出（全ジャンル共通で1回やると効果大）
改造の瞬間に**軽い暗転フラッシュ**（`setVignette` 一瞬1→0）＋**既存客が一斉に"入れ替わりモーション"**（0.4秒で全員 static→新ポーズへ ease）。これだけで「セリフだけ」感が消える。各ジャンル固有はこの上に足す。

## 1. ishiki（意識高い系ビジネス）— 「意識高い朝活カフェ」
色 `0x2F6FDB` / tint `0x8cbeff`。トーン: 朝日・MacBook・覚悟・仕組み化。

### 1. 客のバリエーション（4人）
- 配置: 中央テーブル寄り `[-4,-4]`, `[4,-4]`, `[-4,2]`, `[0,-6]` の椅子。全員 **sit** クリップ（プリミティブなら legL/legR を -1.4、group.y を座面 0.5 に）。
- 小物「MacBox」: `BoxGeometry(0.34,0.02,0.24)` の薄い天板＋`BoxGeometry(0.34,0.22,0.02)` の画面を L字に。画面色 `0xcfe4ff` emissive 弱、リンゴ代わりに白い小 Box を1個。各客のテーブル上 `y≈1.05`、客の正面に置く。
- 服色: 白〜ネイビーのシャツ系（HSL h=0.6, s=0.15, l=0.7〜0.9）。全員こざっぱり。
- 振る舞い/個体差（3〜5パターン、1人1つ）:
  1. **力説マン**: `emote-yes` 連打（腕Rを上げ下げ `armR.rotation.x` を -2.4↔-1.6 で往復、頭は小刻みnod）。「本質的には覚悟なんですよ」感。
  2. **タイピング勢**: 両手を MacBox 上に、`armL/armR` を -1.2 前傾、手先を細かく上下（キー打鍵の微振動）。
  3. **首がっつり頷き**: `emote-yes`、相手のいない方向へ延々と頷く（1人で頷いてる不気味さ）。
  4. **腕組み俯瞰**: `static`＋腕を胸前でクロス（armL/R を内向き z 回転）。時々ゆっくり首を振る（`emote-no`）。
- 追加ネタ: 全員がテーブルではなく**中央平積み棚（＝改造装置）の方をチラ見**する向きに1〜2人。「聖地扱い」。

### 2. 店員の変化
- エプロンはそのまま、**ヘッドセット**を追加: 頭側面に `TorusGeometry(0.14,0.02)` を耳に、口元へ細い Cylinder のマイクアーム。
- 振る舞い: 巡回はするが、立ち止まると `emote-yes`（力説ポーズ）で無言プレゼン。手に**小さいホワイトボード**（`Box(0.4,0.3,0.02)` 白）を `holding-left` で持つ。
- 立ち位置: レジ横 `(-6,0,5)` で1人、客に向かって半身。

### 3. 店内オブジェクト/環境
- **朝日ライティング**: 追加の `DirectionalLight` を低い角度（`position (12, 4, -6)`、色 `0xffe0a0`）で差し込む。壁に強い斜め光＝「朝活」感。既存 hemi は青寄り（applyTint で処理済み）とのコントラストが出る。
- フォグはやや薄め（近 22 / 遠 45）にして"クリアで前向きな朝"。
- **ミニ観葉植物**を各テーブルに（Cylinder鉢＋Coneの葉）。「意識高い店にありがち」。
- 壁に**格言ポスター**: 壁面に `PlaneGeometry(1.6,1.0)` を数枚、`CanvasTexture` で「朝を制する者が人生を制す」等の文字（実装者判断で1〜3枚）。
- メニュー看板の水を **青い光る Box（マインドセットウォーター）** としてレジ上に1本。

### 4. ギャグ/動く演出
- **「朝日」が窓から差すが、時計はどう見ても夕方**（壁掛け時計 Cylinder＋針、針を17:00に固定）。朝活を主張する光と矛盾。
- MacBox の画面に**同じグラフが全員分ずっと右肩上がり**でループ（emissive を明滅させるだけでそれっぽい）。

### 5. 進行/探索小ネタ
- 「マインドセットウォーター」ボトルにEで話す隠しオブジェクト → 会話カウントには入らないが専用ツッコミ「ただの水やないか」。

## 2. shokai（紹介制ビジネス）— 「奥の席で説明会カフェ」
色 `0xE0A400` / tint `0xffc850`。トーン: 紹介者コード・ホワイトボード説明会・ご友人紹介・パイプ椅子の輪。

### 1. 客のバリエーション（5人）
- 配置: **奥の席**（`z=-6〜-4` 帯）に固める。既存椅子を無視して**パイプ椅子の輪**を新規生成（後述プロップ）し、その円周に座らせる。中心 `(4,0,-5)` 半径1.8、5脚を放射配置、全員内向き。
- 全員 **sit**、中央（ホワイトボード）を向く。
- 小物: 客の手元に**冊子**（`Box(0.3,0.02,0.42)` 白＋色帯）を `holding-both`。1人だけ**手を挙げている**（`armR` を真上、質問者）。
- 服色: 黒〜濃紺のジャケット寄り（h=0.62, s=0.2, l=0.25〜0.4）。妙にフォーマル。
- 個体差:
  1. **説明する人**（＝店員側でなく客が1人前に立ってプレゼン）: ホワイトボード前に立ち `emote-yes`＋指し棒。
  2. **深く頷く新規**: `emote-yes` ゆっくり大きく。感化されてる。
  3. **腕組み品定め**: `static`＋腕クロス、たまに `emote-no`（半信半疑）。
  4. **メモ魔**: 冊子を見て手先を細かく（書き込み）。
  5. **勧誘されてる困り客**: 体は輪の外向き（帰りたい）だが首だけ中へ。半笑い。

### 2. 店員の変化
- エプロンに**「紹介者コード入力受付」名札**（胸に黄色 Box＋白帯）。
- 手に**タブレット/クリップボード**（`Box(0.3,0.4,0.02)`）を `holding-both`、入口付近 `(6,0,-1)` に立ち、来る人を輪へ誘導する向き。
- 振る舞い: `interact`（お辞儀＝torsoを前傾 z）を周期的に。愛想が過剰。

### 3. 店内オブジェクト/環境
- **ホワイトボード**（メイン小道具）: `Box(2.2,1.4,0.06)` 白を輪の中心奥 `(4,1.2,-6.5)`。表面 CanvasTexture で「あなたも→自由→権利収入→仲間」の矢印図と円グラフ。三脚は Cylinder×2。
- **パイプ椅子**: `Cylinder`脚4本＋座面 Box、5脚を上記の輪に。既存のカフェ椅子と違う"事務"感。
- 照明はスポット気味（中央に `SpotLight` 下向き）で「説明会ブース」の閉鎖感。フォグやや濃い。
- レジ横に**「※紹介者コードで¥550」の貼り紙**（Plane＋テキスト）。

### 4. ギャグ/動く演出
- ホワイトボードの図が**定期的に更新**され、矢印の先が毎回「上位会員」「ダイヤモンド」等に増殖（CanvasTexture を数秒ごと差替）。
- 一番外の困り客が**じりじり輪から離れようとしては引き戻される**（x をわずかに往復移動、椅子ごと少しずつ、コミカルに）。

### 5. 進行/探索小ネタ
- 隠し「紹介者コード入力欄」オブジェクト（レジのボード）にEで空欄のまま話す → ツッコミ「コード持ってへんて」。

## 3. gossip（週刊誌ゴシップ）— 「喫茶スクープ」
色 `0xE05555` / tint `0xffcd46`。トーン: ひそひそ・暴露・【衝撃】・新聞散乱。

### 1. 客のバリエーション（6人＝3ペア）
- 配置: 既存テーブル `[-4,-4] [4,-4] [-4,2]` に**2人ずつ顔を寄せて**座る。全員 sit だが**互いに顔を近づける**（`leanWhisper`: 各人の head を相手側へ 0.5rad 傾け、torso も内向きに寄せる。2人の距離を通常より近く）。
- 小物: 各テーブルに**週刊誌**（`Box(0.34,0.02,0.48)`＋赤い【衝撃】帯 Box）を開いて置く。1人はスマホ（`Box(0.16,0.28,0.02)` 画面 emissive）を見せ合う。
- 服色: 派手め・雑多（h ランダム, s=0.5, l=0.5）。生活感。
- 個体差:
  1. **口に手を当てて囁く**（armR を口元へ）。「ここだけの話」。
  2. **のけぞって驚く**（torso を後ろへ z 反り、口ぽかん＝頭を上向き）。「マジで！？」。
  3. **スマホ見せつけ**（armR を相手へ突き出す）。
  4. **周囲を警戒**（頭を左右にキョロキョロ＝shake を遅く）。聞かれてないか確認。
  5. **一人だけ関係なくケーキ食ってる**（無関心、static）。良い箸休め。
- ペアで**囁く→のけぞる→また囁く**のループを軽くタイミングずらして。店内が"噂の波"。

### 2. 店員の変化
- エプロンに**取材パス風の首下げ札**（首から `Box(0.2,0.28,0.01)`＋赤帯）。
- 手に**新聞/ゲラ**を `holding-right`。振る舞い: 客テーブルに近づいて `leanWhisper`（店員も噂に加担）。
- 立ち位置: テーブル間をうろつき、時々立ち止まって周囲を見回す（キョロキョロ）。

### 3. 店内オブジェクト/環境
- **新聞散乱**: 床のあちこちに `Box(0.5,0.01,0.7)` の白/黄ばみ Plane を十数枚、ランダム回転で散らす（`z=-6〜6`, `x=-8〜8`）。踏んで歩ける雑然。
- 壁に**「【衝撃】」の赤い見出し切り抜き**を数枚（Plane＋テキスト、赤帯）。
- 照明はやや黄ばんだ蛍光灯色。フォグ薄め。**赤いネオン「SCOOP」**をレジ上に（発光 Plane＋明滅弱）。
- ショーケースに **【衝撃】本日のケーキ**（Cylinderケーキに赤帯）。

### 4. ギャグ/動く演出
- 天井から**吹き出しアイコン（「!?」「…」）**が客の頭上にふわっと出て消える（小 Plane を上下フェード）。噂が可視化される。
- 一枚だけ散乱新聞に**プレイヤー自身の顔らしき見出し**（「謎の客、平積みを改造か」）→ 近づくと読める。

### 5. 進行/探索小ネタ
- 散乱新聞の1枚がEで読める隠し → 「本件、店主は無関係とのこと」等のメタツッコミ。会話3回とは別。

## 4. tetsu（偏り哲学）— 「存在喫茶」

  4. **横向きに固まる**: 椅子に対し体だけ横、首だけ正面。不自然な静止。

### 3. 店内オブジェクト/環境

- 出口ドア風の Plane に**「出口とは何か」の貼り紙**（back壁 `z=-8` 中央）。

### 4. ギャグ/動く演出
- **裸電球だけが唯一動く**＝「この店で動いてるの、電球とお前だけやん」の絵。
- 指さし客の指の先に、近づくと**極小の"?"パーティクル**がぽつんと浮いている（が意味は無い）。

### 5. 進行/探索小ネタ

## 5. shizen（自然派うさんくさ）— 「波動カフェ」
色 `0x4E9A51` / tint `0x6ebe78`。トーン: 月齢・波動・木の実・デトックス・観葉植物。

### 1. 客のバリエーション（4人）
- 配置: 床に**座布団/クッション直座り**（椅子でなく床、`[-4,1] [4,1] [-2,-3] [6,3]`）。sit ポーズを床高（group.y=0）で。あぐら風に膝を開く（legL/R を外へ z 回転）。
- 小物: **数珠風の首飾り**（首に小 Sphere を並べた Torus 状）、手のひらを上に開いて**瞑想**（両 armL/R を前へ、手のひら上向き）。1人は**木の実を摘まんでいる**。
- 服色: 生成り・アースカラー（h=0.1, s=0.3, l=0.55〜0.7）。麻っぽい。
- 個体差:
  1. **瞑想（目を閉じ静止）**: static＋手のひら上、ゆっくり上下呼吸（group.y を微小 sin）。
  2. **手をかざして波動チェック**: 相手やコーヒーに手をかざす（armR 前へ、手先を小さく回す）。
  3. **首をゆっくり回す**（デトックス体操）: head を大きくゆっくり回転。
  4. **裸足で足裏を触る**（アーシング）: 前かがみで足へ（torso bend）。
- 演出: 全員が**やたらゆっくり**。せかせかした ishiki の対極。

## 8. 実装優先度（まず入れると効果大な順）

## 付録: `applyGenreScene(key)` の実装スケルトン（実装者向け）
```
// triggerEvent() 末尾で applyGenreScene(activeGenre) を呼ぶ。
// 改造前のNPC/プロップは genreProps[] に積んで、遷移時に一括除去できるように。
const genreProps = [];
function clearGenreProps(){ genreProps.forEach(o=>scene.remove(o)); genreProps.length=0; }

function applyGenreScene(key){
  commonTransformFlash();          // 0-4: 暗転フラッシュ＋客ease
  swapCustomers(key);              // 各1: 客の入替（配置・pose・小物）
  updateClerks(key);               // 各2: 店員の見た目/振る舞い
  addGenreProps(key);              // 各3: プロップ生成 → genreProps へ
  setGenreAtmosphere(key);         // 各3: 追加ライト/フォグ/浮遊粒
  startGenreGags(key);             // 各4: update() 内で回すギャグの登録
  // chaos は上記各関数の中で key==='chaos' 時に「各ジャンル縮小版」を合成
}
```
NPC のポーズ更新・ギャグ（増殖椅子/UFO移動/明滅/満月）は既存の `update(dt)` ループ末尾に `updateGenreGags(dt)` を1本足して回す。`eventTriggered` 後のみ実行。
