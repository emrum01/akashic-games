# ブックカフェ思想改造ゲーム — 空間モーフ（部屋の"形"変貌）＆ 客の衣装 設計書

対象: `bookcafe-3d.html` / 実装者: three.js エンジニア
前提: 既存 `DESIGN-variations.md`（#0〜#8）・`DESIGN-dialogue-and-overhaul.md`（ラリー＋群衆リビルド＋壁掲示）と整合。既存の #1〜#12（会話ラリー・全ジャンル変貌・性能安全弁・切替/リトライ復元）を壊さない。

ユーザー要望: **「道具が出るだけ」でなく、店の"形"そのものが四角から変わり、改造の瞬間に"別の店になってしまった"最大インパクトが欲しい。客の服装もジャンル別に。**

2本立て:
- **企画1 — 部屋の"形"をジャンル別にリビルド**（建築そのものを変貌させる）
- **企画2 — 客/店員の衣装をジャンル別に**（Kenneyテクスチャ差替＋プリミティブ着衣/小物）

# 0. 既存の空間・衝突モデル（必読・ここに追従させる）

コードから読み取った現状の"真実"。新形状はこれを genre 別に**作り直す**。

### 0-1. 部屋と建築（`buildRoom` @ L845〜）
- `ROOM = {minX:-11, maxX:11, minZ:-8, maxZ:8, wallH:4}`（内寸 22×16、天井高4）。
- 壁は4枚の Box（back z=-9 / front z=+9 / left x=-12 / right x=+12、厚み0.3、高さ4）。床/天井は Plane。
- 内装マテリアルは `interiorMeshes[]` に積まれ `applyTint` で色変更。
- 装飾本棚（右壁）とレジ（front-left）は**視線遮蔽 `occluders[]`（AABB）**として登録。

### 0-2. 衝突（ここが最重要）
- **プレイヤー衝突は矩形 `clampToRoom(x,z,pad)`（ROOM bounds） ＋ 中央棚の半径1.9円** の2つだけ（`updatePlayer` @ L3120〜3154）。
- `occluders[]` は**視線判定専用**で、歩行衝突には使われていない（＝壁以外の物は通り抜ける）。
- カメラも矩形 ROOM bounds でクランプ（L3195〜3197）。
- **結論**: 部屋の形を変えるには「①矩形 bounds の縮小」＋「②内側に**衝突AABB群 `solids[]`** を新設し clampToRoom を"bounds→solids押し出し"に拡張」の2段で表現するのが最小コスト。完全な非ユークリッド/曲線衝突は不要——**AABB群で"部屋の形"を近似**する。

### 0-3. 環境の退避/復元（既に実装済・流用）
`stashEnv()`→`savedEnv{fogNear,fogFar,hemiInt,ambInt}`、`grabAmbient()`、hemi は `scene.userData.hemi`。リトライ/切替で元へ戻す経路あり（L2808〜, L3074〜）。**部屋モーフも同じ思想で「適用→復元」できるように**。

# 企画1: 部屋の"形"をジャンル別にリビルド

## 1-0. 実装フレーム（全ジャンル共通の土台）

### 現実解の宣言（できること/しないこと）
- **する**: 変貌時に既存の4枚壁＋床天井を撤去し、ジャンル別の**壁Box群・天井高・床形・入口/窓・照明/フォグ**を新規生成。矩形bounds を縮小し、内側に衝突AABB群 `solids[]` を置く。
- **しない**: 曲面の厳密衝突、真の非ユークリッド、破壊可能建築。**丸い部屋は「多角形（8〜12角）の壁Box近似」**で見せる。傾き/歪みは**メッシュを回転/せん断して"見た目"で**表現し、衝突は素直なAABBで取る（見た目と衝突が多少ズレても没入は保てる）。

### 追加する2つの仕組み（実装者向け提案）
```
// ① 部屋プロップ置き場（genreProps と分離。壁/床/天井/柱など建築要素）。
const roomProps = [];
function clearRoomProps(){ roomProps.forEach(o=>{try{scene.remove(o);}catch(e){}}); roomProps.length=0; }

// ② 衝突AABB群。clampToRoom を「bounds クランプ後、各solidから押し出し」に拡張。
let solids = [];  // [{minX,maxX,minZ,maxZ, tall?}]  tall=天井まで=視線も遮る
let ROOM_ACTIVE = {...ROOM};   // ジャンル別に縮小した実効bounds（clamp/カメラが参照）
function clampToRoom(x,z,pad){
  pad=pad||0.6;
  let cx=Math.min(ROOM_ACTIVE.maxX-pad,Math.max(ROOM_ACTIVE.minX+pad,x));
  let cz=Math.min(ROOM_ACTIVE.maxZ-pad,Math.max(ROOM_ACTIVE.minZ+pad,z));
  // solids からの最小押し出し（AABB×点、pad込み）
  for(const s of solids){
    if(cx>s.minX-pad&&cx<s.maxX+pad&&cz>s.minZ-pad&&cz<s.maxZ+pad){
      const dl=cx-(s.minX-pad), dr=(s.maxX+pad)-cx, dd=cz-(s.minZ-pad), du=(s.maxZ+pad)-cz;
      const m=Math.min(dl,dr,dd,du);
      if(m===dl)cx=s.minX-pad; else if(m===dr)cx=s.maxX+pad; else if(m===dd)cz=s.minZ-pad; else cz=s.maxZ+pad;
    }
  }
  return {x:cx,z:cz};
}
```
- `ROOM_ACTIVE` はカメラクランプ（L3195）でも参照させる。既存 `ROOM` は"最大外寸"として残し、`ROOM_ACTIVE` を都度書き換え。
- **中央棚の半径1.9円**衝突は全形状で維持（棚は常に部屋中心にある）。棚をずらすジャンル（後述）では円の中心も動かす。
- 変貌解除/リトライ時: `clearRoomProps()`＋`solids=[]`＋`ROOM_ACTIVE={...ROOM}`＋`buildRoom()`再生成 で原状復帰。

### 変貌の"見せ方"（最大インパクトの瞬間）
既存 `commonTransformFlash()`（ビネット1→0）を土台に、部屋モーフ演出を上乗せ。実装しやすい順に:
1. **暗転フラッシュで隠して差し替え**（最も安全・確実）: フラッシュのピーク（真っ白/真っ暗の1〜2フレーム）で `buildRoom` を撤去→新形状を一括生成。プレイヤーには「まばたきしたら別の店」に見える。
2. **壁せり上がり**（0.4〜0.6秒）: 新しい壁Boxを `scale.y=0` or 床下 `y=-4` から生成し、ease で規定高へ迫り上げる。天井は最後に `y` を落として"蓋"をする。ゴゴゴ…の効果音（既存 event sfx 流用）。
3. **天井が抜ける/上がる**: tetsu（虚無の大空間）は天井Boxを一気に `y=4→12` へ上昇させ**吹き抜け**を作る。逆に shokai（個室）は天井を `4→2.6` に下ろして圧迫。
4. **床の張り替え**: 床マテリアル/形を差し替え（木→カーペット→土間→水面風）。1フレームで良い。

> 推奨: **基本は①暗転差し替え**で全ジャンル成立させ、目玉ジャンル（tetsu の吹き抜け / occult の歪み）だけ②③のアニメを足す。全ジャンルにアニメを付けると重い&バグの温床。

## 1-1. ishiki —「天井の高いロフト」（明るく広い吹き抜け）

風刺: 意識高い系が好む"開放的なコワーキング/ロフト"。無駄に天井が高くて明るい。

- **形/寸法**: 矩形は維持しつつ **天井を `4→7` に上げて吹き抜け**、`ROOM_ACTIVE` は元のまま（広さ維持）。中2階風の**ハーフロフト床**を back側（z=-8〜-4, y=3.2）に張り、下から見上げると"階層"がある。
- **壁/建築**: back壁上部を**全面ガラス窓**（大きな半透明青Plane）にして光を入れる。柱を数本（細い白Cylinder, y=0〜7）＝ロフト感。solids に柱4本の細AABB。
- **入口/窓**: front中央にガラス自動ドア風の大開口。窓多め＝明るい。
- **明るさ**: hemi/ambient を **1.3倍**（既存より明るく）＋朝日 DirectionalLight（既存案）。フォグ薄い（近24/遠48）。
- **中央棚/開始/巡回**: 棚は中央維持。プレイヤー開始(0,0,5.5)維持。巡回店員はロフト下と1階を回る（既存ルートでOK、z-6ルートはロフト下）。
- **衝突**: 柱4本を solids に。ロフトは"見た目"のみ（上がれない、天井の演出）。

## 1-2. shokai —「間仕切りのセミナー個室群」（狭い迷路）

風刺: 奥に連れ込む"説明会ブース"。パーテーションで刻まれた狭い区画。

- **形/寸法**: `ROOM_ACTIVE` を **狭める**（例 minX-9/maxX9/minZ-7/maxZ7）＋**天井を `4→2.6` に下げて圧迫**。
- **壁/建築（目玉）**: **パーテーション壁を内部に多数**。例（solids に登録＝実際に通れない迷路）:
  - `{minX:-9,maxX:-2, minZ:-2,maxZ:-1.6}`（横仕切り）
  - `{minX:2,maxX:9, minZ:-2,maxZ:-1.6}`
  - `{minX:-0.5,maxX:0.5, minZ:-7,maxZ:-2}`（縦仕切り）
  - `{minX:5,maxX:5.4, minZ:1,maxZ:7}` 等、3〜4個の小個室を作る。高さ2.2の半端な高さ（覗ける圧迫感）。tall:true で視線も遮る＝店員から隠れやすい導線変化。
- **入口/窓**: 窓を塞ぐ（外が見えない閉鎖感）。入口だけ front-right に細く。
- **明るさ**: 各個室に下向き SpotLight、通路は暗い。フォグやや濃い黄土。
- **中央棚/開始/巡回**: 棚は中央だが**周囲を仕切りで囲む**（"聖域化"）。プレイヤー開始は front-right の入口(6,0,6.5)へ移す。巡回店員は狭い通路を縫う短ルートに差し替え（新 solids に沿う）。
- **衝突**: パーテーション群＝solids の主役。**迷路化で"別の店で道が変わった"感が強い**。

## 1-3. gossip —「雑然と狭い路地裏喫茶」（L字・ごちゃつき）

風刺: 噂の巣窟。物が溢れて狭く、奥まった一角がある。

- **形/寸法（目玉: L字）**: 矩形の**右奥をL字に欠く**。`ROOM_ACTIVE` は minX-9/maxX9/minZ-7/maxZ7 に縮め、さらに **右奥ブロック `{minX:4,maxX:9, minZ:-7,maxZ:-2, tall:true}` を solid で塞ぐ**＝L字の平面形。天井 `4→3`。
- **壁/建築**: L字の欠けた面に**煉瓦/古壁**（テクスチャあれば）。低い棚・積み上げた新聞束（Box塔）を solids に散らして"雑然と狭い"。
- **入口/窓**: 路地裏なので窓は小さく高い位置に1つ、赤ネオン「SCOOP」を外壁風に。
- **明るさ**: 黄ばんだ蛍光灯＋チラつき弱。フォグ薄め。全体に狭く薄暗い。
- **中央棚/開始/巡回**: 棚は中央だがL字で回り込みにくい。開始(0,0,6)。巡回店員はL字通路を回る。
- **衝突**: L字欠き＋新聞束の塔（2〜3個）を solids に。**"道が折れ曲がった別の店"**。

## 1-4. tetsu —「だだっ広い虚無の大空間」（高天井・濃霧）【最大インパクト候補】

風刺: 存在喫茶。無意味に広く、天井が異様に高く、霧で果てが見えない。

- **形/寸法（目玉: 吹き抜け＋拡張）**: **天井を `4→12` へ一気に上昇**（変貌演出で"天井が抜ける"）。`ROOM_ACTIVE` は元のまま or わずかに拡張、だが**濃霧で壁が見えない**＝実質"果てのない空間"に感じる。
- **壁/建築**: 壁を暗くして霧に溶かす（壁マテリアルを背景色に寄せる）。中央に**意味不明な白い大きな "?" モニュメント**（高さ6のオブジェ、solid 円柱）。
- **入口/窓**: 窓なし。back壁遠くに"出口"のドアだけ淡く光る（近づいても霧で遠く感じる）。
- **明るさ（核）**: hemi/ambient を **×0.4**（既存 tetsu 案と同じ）。**フォグ濃く近い（近8/遠22）**＝5m先が闇。唯一の光源＝揺れる裸電球。
- **中央棚/開始/巡回**: 棚は中央。プレイヤー開始(0,0,5.5)。巡回店員は**霧の中からぬっと現れる**（遠くは見えないので巡回が怖い＝ゲーム的にも効く）。
- **衝突**: "?"モニュメントの円柱を solid に。壁は遠くて実質触れない。**天井が抜けて霧に包まれる瞬間が最大の"別世界"演出**。

## 1-5. shizen —「丸いドーム温室」（角を無くす）

風刺: 波動カフェ。角のない有機的なドーム、植物で埋まった温室。

- **形/寸法（目玉: 円形近似）**: **12角形の壁Box近似で"丸い部屋"**。中心(0,0)半径9の円周に、幅3.2の壁Boxを12枚、各30度回転で並べる（`for i in 12: angle=i*30°; wall at (cos*9, sin*9), rot.y=angle`）。天井は**ドーム風**＝低ポリの半球（SphereGeometry 上半分, 半径10, y=0基点）を淡い緑ガラスで。`ROOM_ACTIVE` は使わず、**円クランプ**（`if(hypot(x,z)>8.4) 半径8.4へ丸める`）を clampToRoom に足す（このジャンル専用フラグ）。
- **壁/建築**: 12角の各面に観葉植物を立てかけ、角を植物で隠す＝完全な有機空間。
- **入口/窓**: ドーム全体が半透明で外光（月光）が拡散。入口は1面だけ開口。
- **明るさ**: 緑がかった柔光＋満月ライト。フォグ薄い緑。浮遊粒。
- **中央棚/開始/巡回**: 棚は中央（円の中心＝ちょうど良い）。開始は円周内側(0,0,5)。巡回店員は円周に沿って回る（ルートを円形に）。
- **衝突**: 円クランプ（半径8.4）＋中央棚円。**"四角い店が丸くなった"インパクト大、かつ実装は12枚壁＋円クランプで軽い**。

## 1-6. occult —「歪んだ非対称の異界聖堂」（傾き・不等辺）【最大インパクト候補】

風刺: Bookcafe 異界。まっすぐな線が無く、天井も床も傾き、増殖する。

- **形/寸法（目玉: 歪み）**: 壁を**不等辺・非平行**に。例: left壁を内側に傾け（rotation.z を +0.12）、back壁を高く(6)・front壁を低く(3)、right壁を手前に寄せて台形の平面。`ROOM_ACTIVE` は不等辺を内包する保守的な矩形（minX-8/maxX10/minZ-7/maxZ8）にして、はみ出しは solid の斜め壁近似（階段状AABC数個）で。
- **壁/建築**: 天井を**傾けて**（片側 y=3, 片側 y=6, メッシュを rotation.x で傾斜）。柱を**斜めに**（rotation.z 付き）。まっすぐな線を排除＝眩暈感。
- **入口/窓**: 窓は歪んだ台形。ドアが複数あって1つは"存在しない"（半透明ちらつき）。
- **明るさ**: ちらつき照明（既存 occult 案）＋暗紫フォグ濃い。時々完全暗転→フラッシュ（この瞬間に増殖椅子/幽霊がスポーン）。
- **中央棚/開始/巡回**: 棚を**中心から少しずらす**（不安定感、円中心も追従移動）。開始(0,0,6)。巡回店員は"2体に見える"（既存ドッペル案）＝どっちが本物か歪みで分からない。
- **衝突**: 傾いた壁は"見た目"回転、衝突は階段状AABB近似で取る（見た目とズレてOK＝異界だから）。**傾いた建築＝最も"別の店になってしまった"不気味さ**。

## 1-7. chaos —「断片の無節操な接合」（各ジャンルの部屋の破片を継ぎ接ぎ）

風刺: 思想の闇鍋。ishiki の吹き抜けの隣に shokai の仕切り、gossip のL字欠け、tetsu の "?" 柱、shizen の円弧壁、occult の傾き壁が**同じ部屋に同居**。

- **形/寸法**: `ROOM_ACTIVE` は素の矩形。内部を4象限に割り、各象限に別ジャンルの建築断片を1つずつ置く:
  - 左奥: 天井高7の吹き抜け片＋柱（ishiki）
  - 右奥: パーテーション個室1つ（shokai）＋新聞束（gossip）
  - 左手前: 円弧壁3枚（shizen）＋観葉
  - 右手前: 傾いた壁1枚（occult）＋"?"柱（tetsu）
- **天井**: 象限ごとに高さが違う**ガタガタ天井**（Box を y違いで4枚）。
- **明るさ**: 虹色tint回転（既存）＋朝日＋スポット＋ちらつきが全部混在。フォグ虹追随。
- **中央棚/開始/巡回**: 棚は中央（唯一の"軸"）。開始(0,0,6)。巡回店員は象限を跨いで挙動不審に。
- **衝突**: 各断片の solid を全部盛り（間引いて計6〜8個）。**"店の自己同一性が崩壊した"継ぎ接ぎ建築**。

# 企画2: 客/店員の衣装をジャンル別に

# 3. 実装優先度（インパクト大な順・更新）

部屋モーフは"別の店"感の本丸。テンプレ化して横展開する順。

1. **空間モーフ基盤（1-0）＋暗転差し替え演出＋衝突拡張（solids/ROOM_ACTIVE/clampToRoom）**: これ無しに形は変えられない土台。`roomProps`／`clearRoomProps`／`solids` 押し出し／カメラ ROOM_ACTIVE 参照／リトライ復元まで。**まずここを通すと以降は各ジャンルの壁座標を差すだけ**。
2. **目玉2ジャンルを"形ごと別店"の完成形に**: **tetsu（天井が抜けて霧の大空間）** と **shizen（12角の丸いドーム＋円クランプ）** を推奨。両者は「四角からの脱却」が最も分かりやすく（吹き抜け／円形）、実装は天井上昇＋フォグ、12枚壁＋円クランプで比較的軽い。この2つで"別の店"の手応えと性能を確認＝横展開テンプレ。
3. **衝突ありの形変化（shokai 個室迷路 / gossip L字 / occult 歪み）**: solids で通路が変わる＝ゲーム導線ごと別店。occult の歪みは"見た目回転＋AABB近似"で最大の不気味さ。巡回ルート・開始位置・棚位置の追従もここで詰める。
4. **客/店員の衣装（企画2）**: Kenney tex 割当が確定したら差替＋`dressHumanoid` の小物（帽子/名札/花冠/UFO帽/お札）。群衆リビルドに組み込む。**形が変わった空間に衣装が揃うと"別の店"が完成**。
5. **ishiki ロフト ＋ chaos 継ぎ接ぎ＋変貌アニメの磨き込み**: 残りジャンルと、目玉ジャンルの"壁せり上がり/天井抜け"アニメ（②③）を仕上げに。chaos は各断片の合成なので 1〜3 のテンプレが揃えば組める。

# 付録: 実装者向けフック早見（空間モーフ）

- 建築: `roomProps[]`＋`clearRoomProps()`。変貌で `buildRoom` 撤去→ジャンル別 `buildRoomFor(key)` 生成。復元は `clearRoomProps()`→`buildRoom()`。
- 衝突: `solids[]`（AABB, tall で視線も遮断）＋`ROOM_ACTIVE`（縮小bounds）。`clampToRoom` を bounds＋solid押し出しに拡張。shizen のみ円クランプ分岐。中央棚の半径1.9円は維持（棚移動時は中心追従）。
- カメラ: L3195 の ROOM→`ROOM_ACTIVE` 参照へ。
- 追従: 変貌時に「棚位置／プレイヤー開始／巡回ルート（clerks[].route）」を新形状用に再設定。群衆リビルド（`rebuildCrowd`）は新 solids/形状に合わせ座標を出す。
- 環境: 天井高/フォグ/照明は `setGenreAtmosphere` に集約、`stashEnv`/`savedEnv` で復元（既存流用）。
- 演出: 基本＝暗転フラッシュ差し替え（`commonTransformFlash` ピークで生成）。目玉のみ壁せり上がり/天井抜けを `updateGenreGags` の初回数フレームで ease。
- 衣装: `buildHumanoid(opts.outfit)`＋`dressHumanoid(g,outfit)`。店員は `updateClerkAppearance` 拡張。Kenney tex は `material.map` 差替（素材確定待ち）。同種小物はジオメトリ共有・`reduced` で最小化。
