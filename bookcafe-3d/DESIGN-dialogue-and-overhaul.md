# ブックカフェ思想改造ゲーム — 会話ラリー台本 ＆ 内装抜本リニューアル設計書

対象: `bookcafe-3d.html` / 実装者: three.js エンジニア
前提: 既存 `DESIGN-variations.md`（#0〜#8）と、すでに実装済みの `applyGenreScene / swapCustomers / applyPose / genreProps / updateGenreGags / commonTransformFlash` フレームワークの上に足す。既存の全ジャンル変貌の枠組みは維持。原作ジャンルデータ（ishiki/shokai/gossip/tetsu/shizen/occult/chaos＋normal, 各 clerk/cust/menu/tsukkomi）を尊重し、**既存テキストは消さず、ラリーの一部として引用・拡張**する。

この設計書は2本立て:
- **企画1 — 会話ラリー台本**: 一言発話を、Eを押すたびに1行進む多往復のラリーへ。プレイヤーの相槌/ツッコミも混ぜる。無制限ループ前提。
- **企画2 — 内装の抜本リニューアル**: 小道具追加でなく"別の店"に。客の人数・構成、小道具物量、壁掲示、床壁照明フォグ色まで。

# 企画1: 会話の「ラリー」台本

## 1-0. 会話モデル（実装仕様）

### 現状と変更点
現状 `onTalk()` は「1回押す＝ランダム1行」。これを **NPCごとに会話シーケンス（ラリー）を持ち、Eを押すたびに1行ずつ進む**方式へ変える。会話数で強制クリアしない設計なので、ラリーは**何度でも繰り返せる**。

### データ構造
話者は `cust` / `clerk` / `player` の3種。ラリーは「話者＋セリフ」の配列。
```
// genre → 話者役割 → ラリー配列（複数パターン）。各ラリーは行の配列。
// 行 = { who:'cust'|'clerk'|'player', line:'…' }
const RALLIES = {
  ishiki: {
    cust:  [ [ {who:'cust',line:'…'}, {who:'player',line:'…'}, … ],  // パターンA
             [ … ] ],                                                 // パターンB
    clerk: [ [ … ] ]
  },
  … normal含む全ジャンル …
};
```

### NPC への割当と進行
- 各 NPC（`npcs[]` の要素）に `rally`（パターン1本を選択）と `idx`（現在行）を持たせる。改造時／初期化時に `pickRally(genre, kind)` でパターンをランダム選択。
- `onTalk()` で対象NPCの `rally[idx]` を表示 → `idx = (idx+1) % rally.length`。**末尾まで行ったら先頭へループ**（＝無制限に続く）。1周したら別パターンへ差し替えても良い（`idx===0 に戻る時に pickRally 再抽選`）。
- **話者の出し分け**: 行の `who` によって吹き出しの名札を「お客」「店員」「あなた」に。`player` 行は吹き出し色を変える（プレイヤー視点の内心/ツッコミ）。`showBubble(whoLabel, line)` に色引数を足すだけ。
- **クリア判定**: 既存の「会話3回でクリア」を撤廃済みなら、ラリーを1つのNPCで**1周完了**したらそのNPCの頭上に小チェック（見て回った実感）。強制はしない。
- **相槌行の間**: player 行は少し早めに自動送りにしても良い（0.8秒後に次へ促すヒント）。基本はE手動送りで良い。

### トーン方針
- **cust/clerk 行 = 原作の胡散臭い本人たち**。原作既存セリフを**起点や決め台詞**として温存し、その前後に接続してエスカレーションを作る。
- **player 行 = 乾いた関西弁ツッコミ / 内心**。声に出さない `（…）` 表記で"心の声"、たまに口に出る一言。原作 tsukkomi のトーン。
- ラリーは**尻上がりにやばくなって**、最後にプレイヤーの決めツッコミか、相手のさらに斜め上の一言で締める。締めた後は先頭へループ（何度でも味わえる）。

# 企画2: 内装の抜本的リニューアル（"別の店"化）

## 2-0. 方針と共通仕組み

### 既存フレームへの接続
既存の `applyGenreScene(key)` が呼ぶ `swapCustomers / updateClerkAppearance / addGenreProps / setGenreAtmosphere / startGenreGags` を**強化**する。特に:
- `swapCustomers` は現状「既存4体を再ポーズ」だけ。→ **`rebuildCrowd(key)` に格上げ**: 既存 npcs を一旦除去し、ジャンル指定の**人数・組み方・座標・ポーズ・ラリー**で作り直す（下記各ジャンルの表通り）。人体は既存 `buildHumanoid`（GLB があれば差替）。
- 小道具は全部 `genreProps` に push（遷移時 `clearGenreProps` で一括除去、既に実装済み）。
- 壁掲示は `makePoster(text, w,h, colorBg, colorFg)`（`CanvasTexture` を Plane に）を1個作れば全ジャンル使い回せる。**素材調査で CC0 ポスター画像が来たらそれを貼る、無ければ CanvasTexture で文字を描く**。

### パフォーマンス指針（人数増やす前提の安全弁）
- 群衆は**最大 ~14体/ジャンル**を上限目安。低スペック時は既存 `reduced` フラグ（`buildStaticNPCs` の npcN 参照あり）で半減。
- 会話ラリーは近接NPCのみ更新。ポーズアニメは可視かつ near のみ。
- 小道具は"物量に見せる"のが目的なので、**同一ジオメトリ/マテリアルを共有**（InstancedMesh 推奨: 椅子・鉢・新聞・本など同種を大量に）。

### 素材（CC0）活用の分岐
別エージェントが収集中。来たものに応じて:
- **壁/床テクスチャ** → `tryTexture()`（既存ヘルパ, `assets/tex/`）に置くだけで内装マテリアルへ反映。ジャンルごとに床/壁テクスチャを差し替え。
- **ローポリ小道具 GLB**（観葉植物・椅子・UFO・本の山など）→ そのジャンルの `addGenreProps` で N個 `clone()` 配置。無ければ本設計のプリミティブ案。
- **追加キャラ GLB** → `buildHumanoid` の代わりにロード、`applyPose` 相当をボーン/クリップで。

## 2-8. 実装優先度（更新版・効果大な順）

これまでの #8 を、ラリー＋群衆リビルドを含めて更新。

1. **会話ラリー基盤（1-0）＋ normal/1ジャンルの台本投入**: `onTalk` をラリー方式に変え、NPCに rally/idx を持たせ player 行の吹き出し色を分ける。まず normal＋ishiki の台本を入れる。**"一言→多往復"の体感変化が最も大きく、テキストだけで実装が軽い**。効果対コスト最強。
2. **`rebuildCrowd(key)`（群衆リビルド）**: swapCustomers を「4体再ポーズ」から「ジャンル指定の人数・組み方・座標・ポーズ・ラリー割当で作り直す」へ格上げ。**"別の店"感の本体**。まず shokai（3人組×3）か gossip（囁きペア×4）で"組"の面白さを立てる。
3. **小道具物量＋壁ポスターの `makePoster` 基盤**: `makePoster` 1関数＋ InstancedMesh で同種小物を大量配置。1ジャンルを"空間ごと別店"レベルまで作り切る（推奨: shokai のブース化 or shizen の密林化＝物量が映える）。素材が来たらここでテクスチャ/GLB を差す。
4. **床/壁/照明/フォグの抜本変更**: tetsu の暗転＆濃霧、ishiki の朝日、occult のちらつきなど"空気ごと別店"。既存 stashEnv/savedEnv で復元可能。ジャンルの世界観を照明で決める。
5. **残りジャンルの台本＆群衆＆小道具を横展開 → chaos 全部乗せ**: 2〜4 のテンプレを各ジャンルに展開し、最後に chaos で合成。ギャグ（切替看板・UFO・増殖椅子・幽霊消滅・逆回転時計）を仕上げに散らす。

## 付録: 実装者向けフック早見

- ラリー: `RALLIES[genre][kind]` = パターン配列、各パターン = 行配列。`onTalk` で対象NPCの `n.rally[n.idx]` を出し、`n.idx=(n.idx+1)%len`。1周で再抽選可。`player` 行は名札「あなた」＋色違い吹き出し。無制限ループ。
- 群衆: `rebuildCrowd(key)` — 既存 npcs 除去 → 表の人数/座標/ポーズ/ラリーで `buildHumanoid`（or GLB）生成 → `applyPose` → `n.rally=pickRally(...)`。`reduced` 時は半減。
- 小道具: すべて `genreProps.push()`（`clearGenreProps` で一括除去済）。同種大量は InstancedMesh。壁は `makePoster(text,w,h,bg,fg)`（CanvasTexture, 素材あれば画像）。
- 空気: `setGenreAtmosphere(key)` で fog 近遠・hemi/ambient 強度・追加ライトを変更（`stashEnv`→`savedEnv` で復元）。
- ギャグ: `updateGenreGags(dt)` に登録（増殖椅子/UFO移動/明滅/満月満ち欠け/切替看板/幽霊フェード）。`eventTriggered` 後のみ実行。
