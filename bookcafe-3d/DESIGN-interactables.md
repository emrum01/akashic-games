# ブックカフェ思想改造ゲーム — Eで調べる"物"＋音アイテム＋画像密度 設計書

対象: `bookcafe-3d.html` / 実装者: three.js エンジニア
前提: 既存 `DESIGN-variations.md` / `DESIGN-dialogue-and-overhaul.md` / `DESIGN-space-morph.md` と整合。既存 #1〜#13（会話ラリー[**player行は削除済＝cust/clerkのみ**]・空間モーフ・衣装・リビール等）を壊さない非破壊前提。

ユーザー要望: **画像素材をふんだんに入れてリアリティある空間に／E＝アクションに統一し"物"にEしてもコメントが出る／効果音・音楽が出るアイテムも。**

重要な区別（ユーザー確認済）: **今回の"物へのコメント"は会話ラリーのplayer行とは別物**。オブジェクトを調べたときの短い**フレーバー説明/観察コメント（システム寄りの声）**で、「それが何か／どういう店か」が伝わる短文。**プレイヤーに代弁させない**（ツッコミ調はOKだが一人称の相槌にはしない）。表示は吹き出しではなく**独立した"調べた"表示**（下記 1-0）。

3本立て:
- **企画1 — Eで調べる"物"とコメント**（NPCと物、近い方を対象）
- **企画2 — 音が出るアイテム**（sfx/短音楽ループ）
- **企画3 — 画像素材で密度/リアリティを上げる配置**

# 0. 既存のE/音/座標モデル（必読・ここに追従）

コードから読み取った現状。新機能はこれに接続する。

### 0-1. Eの現状（`onTalk` @ L2393）
- E＝`onTalk()`。`npcs[]`（静的客/店員）＋`clerks[]`（巡回店員）の**最寄り** `DIFF.talkRange=2.6` 内を対象に、ラリー1行を吹き出し表示。無ければ「近くに話せる人がいない」。
- 会話ラリーは `RALLIES[genre][kind]` の実装済（**player行なし**）。`showBubble(who,txt,{player})`／`addLog`／`showHint` あり。
- 別キー: Space＝拾う(`DIFF.pickupRange=1.7`)／平積み棚(`DIFF.tableRange=2.4`)。**Eはアクション統一のため"物調べ"も引き受ける**（ユーザー要望）。

### 0-2. 音の現状（`Audio2` @ L559〜718）
公開API: `Audio2.sfx(name)` / `Audio2.bgm(kind)` / `Audio2.stopBgm()` / `Audio2.heartbeat(vol)` / `beep(freq,dur,type,gain,sweepTo)`（内部）。
- `sfx(name)`: `ASSETS.sfx[name]` のファイルが読めればそれ、無ければ `synth[name]()` にフォールバック（footstep/pickup/place/heartbeat/alert/bust/event/clear/talk が既存）。**新規sfxは `ASSETS.sfx` にファイル名を足す＋`synth` に合成フォールバックを1つ書く**だけで鳴る。
- `bgm(kind)`: `curBgm===kind` ならno-op、違えば `stopBgm()`→ファイル(`ASSETS.bgm[kind]`)or合成pad。現状 `cafe`/`tension` の2種。
- **音アイテムはこの2系統に相乗り**: 単発SFXは `sfx()`、音楽切替は `bgm()`（後述の一時上書き/復帰）。

### 0-3. 座標（物の当たり判定を置く位置）
- レジカウンター `(-7.5, ~1, 5.6)`／本棚（右壁）`x≈10.4, z=-4/0/4`／通常テーブル `[-4,-4][4,-4][-4,2][7,2][0,-6]`／中央平積み棚 原点(0,0)半径~1.9／プレイヤー開始(0,0,5.5)。
- ジャンル別プロップは全て `genreProps[]`（改造後のみ存在）。共通プロップ（レジ/棚/本棚/テーブル/メニュー黒板）は改造前から存在。

# 企画1: Eで調べる"物"とコメント

## 1-0. 実装フレーム

### E の対象拡張と優先順位
`onTalk()` を **`onAction()`** に拡張（名前は任意、Eバインドはそのまま）。近い方を対象に:
1. **最寄りNPC**（`talkRange=2.6` 内）までの距離 `dN`
2. **最寄り調べ物**（各物の `radius` 内）までの距離 `dO`
3. `dN <= dO` ならNPC会話（既存ラリー）、そうでなければ**物のコメント表示**。
4. どちらも無ければ「近くに調べられるものがない」。
- **NPC優先の微調整**: 人と物が同距離帯なら人を優先（会話が主役）。物は「人が居ない一角」で拾わせる設計に自然になる。

### 調べ物レジストリ
```
// 物ごとに: 当たり判定(位置＋半径)、コメント配列(通常/改造後)、任意でsound。
// mesh に紐づけず「座標＋半径」で判定（プロップ再生成に強い）。genre別は eventTriggered で出し分け。
const INTERACTABLES = {
  register: {                       // レジ（共通・改造後で変化）
    pos:[-8.5,1.4,5.6], radius:1.8,
    normal:['古いレジスターだ。手打ちで、レシートに手書きの一言が付く。','引き出しに常連さんの忘れ物メモ。'],
    byGenre:{ ishiki:['…', '…'], gossip:['…'] , /* 各ジャンル 1〜3本 */ },
    sound:'cash'                    // Eで鳴る（企画2）
  },
  // …以下 共通＋ジャンル別を列挙…
};
```
- 各物に**コメント複数本**（繰り返しEで順送り or ランダム）。会話ラリー同様、何度でも楽しめる。
- `byGenre[activeGenre]` があれば改造後はそれを優先、無ければ `normal`。改造前は常に `normal`。
- 進行: 物ごとに `idx` を持ち `(idx+1)%len` で順送り。1周でランダム再抽選も可。

### 表示（会話と区別）
- **物コメントは吹き出し(who付き)でなく、`showInspect(text)`** ＝画面下部などに「― <text>」形式の中立表示（会話バブルと色/形を変える）。名札は出さない（誰の声でもない＝システム観察）。既存 `showBubble` を複製して who を消したものでOK。
- `addLog('◇ '+text)`（会話は「」、調べ物は◇で区別）。

### トーン
- **「それが何か／どういう店か」が伝わる短い観察**。乾いた風刺は可、但し一人称ツッコミにしない。例: ×「なんやこれ」→ ○「"マインドセットウォーター"と書かれた、ただの水。」。事実＋一滴の皮肉。

# 企画2: 音が出るアイテム

## 2-0. 実装フレーム（既存 Audio2 に相乗り）

### 単発SFX（Eで1回鳴る）
- `ASSETS.sfx` に新規キーを追加（ファイルは素材エージェント収集中のCC0を配置）。**ファイルが無くても鳴るよう `synth[name]` に `beep()` 合成を1つ書く**（既存パターン踏襲）。
- 新規SFXキーと合成フォールバック案:

| キー | 用途/物 | 鳴らし方 | 合成フォールバック（beep案） |
|------|---------|----------|------------------------------|
| `cash` | レジ | 1回 | 高short2連（チャリン）: beep(1200,0.05,'square',0.3)＋beep(1600,0.06) |
| `bell` | 呼び鈴 | 1回 | beep(2000,0.4,'sine',0.3, 2000)（チーン、減衰） |
| `brew` | コーヒーマシン | 1回(2秒) | ノイズ風＋低音（シューッ）: 短いsawを揺らす |
| `page` | 本/新聞/冊子 | 1回 | 短いノイズburst（ぺらっ） |
| `key` | MacBox | 1回(連打) | beep(500,0.02)×数発 ランダム音程 |
| `marker` | ホワイトボード | 1回 | beep(300,0.15,'sawtooth',0.2, 380)（キュッ） |
| `chime` | 柱時計 | 1回 | beep(660,0.6,'sine',0.3,660)＋倍音（ボーン） |
| `wind-chime` | 風鈴(shizen) | 1回 | 高音3連ランダム(sine, 減衰)（涼しげ） |
| `neon` | 赤ネオン(gossip) | トグル(ジー) | 低いsawの持続、Eで on/off |
| `drone` | お札/謎箱/増殖(occult) | 1回(不穏) | 低いsine持続＋わずかにデチューン（ゾワッ） |
| `ufo` | UFO(occult) | 1回 | 上下スイープのsine（ヒュゥゥ） |

### 音楽アイテム（BGM切替系）
- **ジュークボックス/スピーカー**を各ジャンルに1つ置く（プロップ）。Eで**そのジャンルのテーマに一時上書き**。
- 方式（実装可能な現実解）: `Audio2.bgm(kind)` は `curBgm` ガードがあるので、**新 kind を追加**（例 `juke_ishiki`）して `ASSETS.bgm` に短いループを登録、合成pad にジャンル別音色を足す（既存 `synthBgm(kind)` の notes/type をジャンルで分岐）。
  - **上書き/復帰**: ジュークにEで `bgm('juke_'+genre)` に切替→もう一度Eで元（改造後は `tension`）へ戻す**トグル**。または「近くを離れると自動で元へ」。実装が軽いのは**トグル**。
  - 重ねる（BGMの上にSFXループ）より**切替**が既存構造に素直（`bgm`は1本しか鳴らない設計）。
- ジャンル別テーマの音色方針（`synthBgm` 分岐）:
  - ishiki=明るい上昇アルペジオ（前向き過剰）／shokai=淡々ループ（洗脳的）／gossip=せかせかした軽音／tetsu=無音に近いドローン（間）／shizen=ヒーリング風sine＋風鈴／occult=不協和音ドローン＋逆再生感／chaos=全部を数秒ごとに切替（音楽版闇鍋）。

### 音の作法（うるさくしない）
- 単発SFXは `volume 0.7` 既定、連打防止に**同一物は0.3秒クールダウン**。
- 音楽トグルは**同時に1本**（`bgm`が保証）。ジュークOFFで改造後BGM(`tension`)へ復帰。
- `drone`/`neon` のループ系はプロップ非表示/改造解除時に必ず停止（`stopBgm`同様のstopをSFX側にも用意 or トグルOFF）。

# 企画3: 画像素材で密度/リアリティを上げる配置

## 3-0. 実装フレーム
- **貼る手段は2つ**: ①`makePoster(imgOrText, w,h)`（`CanvasTexture` に文字 or CC0画像テクスチャを貼った Plane。素材あれば画像、無ければ文字生成）②既存 `tryTexture(url, mat)`（床/壁/テーブル天板のマテリアルに画像）。
- **本の表紙/食品/メニューは小Planeテクスチャ**として棚・テーブル・ショーケースに敷き詰める＝"画で密度"。
- 全て `genreProps`（改造後）or 共通（改造前から）に登録。素材が来なければCanvasTexture生成で成立（ユーザー許容: ローポリ＋実画像混在OK）。

## 3-2. 密度を出すコツ（実装）
- **同種画像はテクスチャatlas＋InstancedMesh**（本の表紙・新聞・お札など大量物）。数十枚でも軽い。
- ポスターは**壁の"F/Z字目線帯"（y=1.4〜2.4）に集中**（プレイヤー視点で目に入る高さ）。
- 実画像とCanvasTexture文字を**混在**させて手作り感（ユーザー許容）。全部を実画像で揃える必要はない。
- **調べ物（企画1）と画像を紐付け**: ポスターにもEコメントを付けられる（1-2表の「格言ポスター」等）＝画像＋コメント＋音の三点で密度が跳ねる。

# 4. 実装優先度（効果大な順）

1. **E統一＋物コメント基盤（1-0）＋共通の調べ物（1-1）**: `onTalk`→`onAction` にして「NPCと物、近い方」を判定、`INTERACTABLES` レジストリ＋`showInspect` で共通物（レジ/メニュー/棚/本棚/時計）にコメント。**"物にEしてもコメントが出る"体感をまず全店共通で成立**。テキストのみで軽く、要望の核。
2. **画像ポスター基盤 `makePoster`（企画3-0）＋1〜2ジャンルの画像密度**: `CanvasTexture`/画像テクスチャで壁ポスター・本表紙・食品を貼る土台。まず ishiki（グラフ/標語）か gossip（新聞/スクープ）で"画で語る密度"を確認＝横展開テンプレ。素材待ちの部分は文字生成で先行。
3. **音アイテム単発SFX（企画2-0 SFX表）**: `cash/bell/brew/page/chime` 等を `ASSETS.sfx`＋`synth`合成で追加し、1-1/1-2の物のEに紐付け。**調べる→音が鳴るの手応え**。ファイル無しでも合成で成立。
4. **ジャンル別調べ物＋その画像＋その音（1-2〜1-8を横展開）**: 目玉ジャンル（occult=お札/UFO/謎箱＋drone/ufo音＋星図・お札画像、shizen=満月/風鈴＋wind-chime＋ボタニカル）から。三点セット（物コメント＋画像＋音）で"リアリティある空間"が完成。
5. **ジュークボックス＝音楽切替（企画2 音楽）＋chaos全部乗せ**: `synthBgm` ジャンル分岐＋トグル切替。最後に chaos で全断片＋切替看板＋音楽闇鍋を合成。

# 付録: 実装者向けフック早見

- E統一: `onTalk`→`onAction`。`dN`(最寄りNPC)と`dO`(最寄り物)を比較、`dN<=dO`で会話、else `inspect(obj)`。同距離はNPC優先。
- 物: `INTERACTABLES[key]={pos,radius,normal:[…],byGenre:{genre:[…]},sound}`。`idx`順送り、`eventTriggered`で `byGenre[activeGenre]`優先。表示は `showInspect(text)`（who無し・色違い）＋`addLog('◇ '+text)`。物は座標判定（メッシュに依存しない＝プロップ再生成に強い）。
- 音: 単発は `Audio2.sfx(name)`（`ASSETS.sfx`にファイル名＋`synth[name]`合成を追加）。同一物0.3sクールダウン。音楽は `Audio2.bgm('juke_'+genre)`（`ASSETS.bgm`＋`synthBgm`分岐）、トグルで `tension`へ復帰。ループ系(drone/neon)は解除時 stop 必須。
- 画像: `makePoster(imgOrText,w,h)`（CanvasTexture 画像 or 文字）／`tryTexture(url,mat)`（床壁天板）。大量物は atlas＋InstancedMesh。ポスターは y=1.4〜2.4 帯に。全て `genreProps`（改造後）or 共通（改造前）へ。
- トーン: 物コメント＝中立の観察（事実＋一滴の皮肉）。**player代弁にしない**。会話ラリー（player行なし）とは別系統。
- 非破壊: #1〜#13 と衝突しない。改造解除/リトライ時に `INTERACTABLES` の byGenre は eventTriggered=false で自動的に normal へ戻る（状態を持たせない）。
