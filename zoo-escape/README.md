# まっすぐ読むな｜動物園脱出

閉園後の動物園から脱出するモバイルファースト RPG 風アドベンチャー Web 謎。
静的 HTML/CSS/JS のみ。ビルド不要。公開想定 URL: https://games.escape-safari.com/zoo-escape/

## ファイル構成

- `index.html` … ゲーム本体（HTML + inline CSS + inline JS）
- `analytics.js` … GA4 計測ラッパー（`window.AkashicAnalytics`）
- `assets/` … 圧縮済み背景 JPEG（幅1280 / 品質80）とキャラ透過 PNG（幅600）

## 遊び方（進行フロー）

1. 導入: 「動物園に迷い込んでしまった。出口を探そう。」
2. 中央広場の看板「まっすぐ読むな」。最初から全文（4行）を読め、**開始直後から解ける**（ライオン前提のロックは撤廃）。看板ビューで端末を左に55度以上・4秒キープすると余分な文字が滑り落ちる。
3. ライオンは「解くための鍵」ではなく **気づくための伏線＋巡回の最初の目的地**。肉やり（傾きチュートリアル）と演出・GA4イベントは維持。看板より先に触れれば「傾ける」に気づく手がかりになる。
4. 探索: 鳥エリア・爬虫類エリアは雰囲気づくりのみ。
5. 残った文字＝巡回順（ライオン→とり→はちゅうるい→でぐち）。看板を先に解いても、巡回で最初にライオンを通るので進行は破綻しない。
6. 巡回: 正しい順でエリアに入ると足跡が光る。順番を間違えるとリセット。
7. 出口ゲート開放 → クリア。

**2つのルート**: (a) 従来＝ライオンで傾きに気づく→看板を解く→巡回→クリア、(b) 新＝開始直後に看板を傾けて解く→巡回（最初がライオン）→クリア。どちらでもクリア可能。

### 事故クリア対策（相対傾き判定）

看板ビューを開いた瞬間の端末角を基準0度とし、そこから **さらに左へ** 傾けた量で判定する（`signLeftTilt()`）。寝転がりプレイや机置きで `gamma` が常時大きい端末でも、開いた姿勢のままでは 0 のままなので意図しないクリアを防ぐ。落下進行は看板ビューを開いている間のみ（エリア画面では進まない）。

## 看板の文面設計（コア謎のキモ）

看板は「夜の動物園の、少し不気味だが自然な4行の注意書き / 詩」として読める。
端末を左に傾け続けると、糊付けの弱い＝余分な文字だけが重力で滑り落ち、
各行に残るのは巡回順を示す文字列になる。「まっすぐ読むな」＝
そのまま（全文を）読むと詩に見えるが、傾けて落として残った文字こそが本当の道順、という二重構造。

| 行 | 落ちる前の全文（＝そのまま読むと自然な詩） | 落ちる文字 | 残る文字 |
|----|--------------------------------------------|-----------|---------|
| 1 | よるの**らいおん**はおきる（夜のライオンは起きる） | よ・る・の・は・お・き・る | **らいおん** |
| 2 | **とり**はそらへかえる（鳥は空へ帰る） | は・そ・ら・へ・か・え・る | **とり** |
| 3 | **はちゅうるい**はねむらない（爬虫類は眠らない） | は・ね・む・ら・な・い | **はちゅうるい** |
| 4 | そこが**でぐち**だ（そこが出口だ） | そ・こ・が・だ | **でぐち** |

残った文字を上から読むと **「らいおん / とり / はちゅうるい / でぐち」** ＝ 巡回順。

### 設計意図

- 落ちる前の4行は、それ単体で「夜の動物園で動物たちが起きている」という不気味かわいい詩として自然に読める（＝ミスリード）。
- 各行の「主語となる動物名（＋出口）」だけが糊が強く残り、述語・助詞側が滑り落ちる。残る文字列が動物名そのものなので、順番として明確に読める。
- 巡回順（ライオン→鳥→爬虫類→出口）はエリア構成と一致し、かつライオンチュートリアルで学んだ「傾ける」操作を看板で再利用させる導線になっている。

## 傾き入力

- DeviceOrientation API（`gamma` で左右傾き）。iOS 13+ は `DeviceOrientationEvent.requestPermission()` が必要なため、ゲーム開始時（タイトルの「はじめる」タップ＝ユーザージェスチャー起点）に一度だけ許可を要求する。許可ダイアログ周りの案内は「このゲームは端末のセンサーを使用します。」という中立表現に留め、解法（傾ける／解ける）は匂わせない。許可されればセンサーで傾きを取得。拒否／非対応の場合は「指の操作だけで進められる」旨を伝えて続行できる。`requestPermission` が無い環境（Android 等）では黙ってリスナー登録する。
- センサー不可 / PC 向け代替: 看板・肉を左へ大きくドラッグ（ポインタ操作）で傾き相当の入力になる。
- `?debug=1` でデバッグ HUD（現在の傾き角・内部カウント）を表示。キーボード左右矢印キーで傾きをシミュレートできる（ヘッドレス検証用）。

## ヒント設計（親切すぎない・自力で気づく体験を守る）

操作指示や答えを UI に直接書かない。ライオンエリアでの傾き体験と看板の文言（「まっすぐ読むな」）から、プレイヤー自身が「傾ける」を連想する。ヒントは詰まった後にだけ段階的に後出しする。

- **ライオンエリア（傾きチュートリアル）**: 「肉が落ちているが、手では届かなそうだ。」まで。「傾けよう」等の指示は出さない。閾値をゆるく（左25度）してあるため、端末を少し動かす／肉を左へドラッグすれば自然に滑って成立する（このゆるさはライオン側のみ）。
- **看板画面**: 詩の全文と雰囲気テキストのみ（開始直後から解ける）。開いた時は「さっきまで微動だにしなかった文字が、いまはなぜか、ぐらついて見える。」——操作方法は書かない。
- **段階的ヒント（詰み防止／先出し禁止）**:
  1. 傾けたが弱い → 第1ヒント「少し傾けるくらいでは足りないようだ。」
  2. 弱い傾きの失敗が重なる（計3回以上） → 第2ヒント。ライオン攻略済みなら「ライオンの肉は、あんなに大きく滑って転がっていった……。」（体験の想起）、未攻略なら「この動物園では、何もかもが少しずつ傾いている……もっと大胆に。」（世界律へ寄せ、ライオン前提にしない）
  3. 看板の前で無操作が続く（約15秒） → 探索へ誘導「看板の前で立ち尽くしていても始まらない。園内を、見て回ろう。」（ライオン等の伏線に気づかせる）
  4. 途中まで落として緩めた場合のみ → 「文字が途中で止まってしまった。落ち切るまで、傾け続ける必要がありそうだ。」（失敗後フィードバックとして維持）
  5. センサー非対応／拒否の環境に限り、看板で複数回失敗した後にだけ代替操作（長押しドラッグ）を案内する。
- **ネタバレ抑制**: 文字が落ちた後も順番（らいおん→…）を読み上げない。「残った文字が、何かの順番に見える。」に留め、実際の順番はプレイヤーが看板の残り文字を読んで気づく。出口ゲートも「看板に残った文字が示す順に回れば」とだけ言い、順番自体は明示しない。

## 状態保存

- 進行状態は `localStorage`（キー `zoo-escape:v1`）に保存。リロードで継続。タイトルの「はじめから」でリセット。

## GA4 計測

`analytics.js` の `window.AkashicAnalytics.track(event, params)` を通じて送信。
`GAME.id = "zoo-escape"` / `GA4_ID = "G-F53JLZ0Z44"`。
イベント: `game_start` / `area_enter` / `meat_delivered` / `sign_solved` / `patrol_complete` / `game_clear`。
`localhost` 等では `isLive=false` となり送信せず、`?debug=1` で console にログ出力。

## 中央広場の看板表示（焼き込み文字の除去＋実テキスト重畳）

生成画像の看板に巨大な「まっすぐ読むな」が焼き込まれていて AI 臭が強かったため、次の方針に変更した。

- **焼き込み文字を除去**: `bg_central_sign.jpg` の看板の板面を無地の灯りのついた木の板にした。手段は ImageMagick（`codex` CLI は導入済みだが画像生成モデルは持たないため未使用）。板面のすぐ上のきれいな照らされた木部を横方向モーションブラーで引き伸ばし、板面全体（画像内 x=534〜794 / y=261〜320 付近）へ被せて文字を消し、周辺の照明グラデーションを保った。夜景なのでわずかな塗り跡は許容範囲。元画像は `/tmp/bg_central_sign_ORIG.jpg` に退避。
- **実テキストを HTML/CSS で重畳**: ゲームが実際に使う 4 行の詩を `#board-text`（明朝系フォント・墨字色）で板面に重ねる。その下隅に落書き風メモ「──まっすぐよむな」を小さく傾けて置く（謎の中心となる違和感なので消さず、ただし控えめに）。
- **板面への正確な配置**: 背景は全エリア共通 1280×720 を `cover` 相当で描画するため、`cover` のスケールとオフセットを JS（`layoutBoardText()`）で計算し、板面の画像内比率 `FACE = {x:0.416, y:0.360, w:0.203, h:0.095}` を画面座標へ変換して重畳。`resize` で追従するので画面サイズが変わっても板面からズレない。遠景では文字は小さく読めなくてよい（読むには看板をタップして調査ビューへ）。
- 看板調査ビュー（傾けて解くモーダル）は従来どおり HTML 描画。遠景の板面テキストと詩の内容は一致させてある。
