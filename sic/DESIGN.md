# 『原文ママ』（sic）体験仕様書 — DESIGN.md

本ファイルは体験仕様の「現在形の正」（G-013）。世界律・真相・オチの結論と、ゲーム内テキスト全文をここに集約する。
「なぜ今そうなっているか」の経緯は `ADR.md`、企画の正は `企画.md`。実装と食い違ったら本書へ寄せて即整合する。

- 表記凡例: **偽文**＝画面に表示される定型文（コピーせず読むと見える文字）。**真文**＝20文字以上でコピーしたときにクリップボードへ注入される、置換前の原文。
- ゲーム内では世界観を説明しない。**偽文は各画面1文以内の含みに留める**。核・真相の全文は本書だけに書く（G-008 差し替え耐性）。
- 真文（SOS等）は制作中に差し替わる前提。**偽文側に真相の結論を書き込まない**。

## 1. 世界観の核と、真相・オチの全文

### 1.1 世界律の核（企画書から）

> **この会社では、画面に表示される言葉はすべて検閲済みである。検閲は担当者が書いた本音の文章を、定型テンプレートへ置換して表示している——書き写された文字だけが、置換前の原文のまま外へ出る。**

検閲は「表示層で本音を定型テンプレートへ置換する処理」であり、**文字データそのものは消していない**。だからコピー（データ層に触れる唯一の行為）だけが置換前の原文を運び出せる。読む＝偽物、写す＝本物。

### 1.2 真相・オチの全文（結論。ゲーム内には出さない）

- プレイヤーが読んでいるのは、菓子メーカー **株式会社コハク製菓** の謝罪文ページ。定番ラムネ菓子「こはくラムネ」の景品キャンペーン（後述）が炎上し、謝罪に追い込まれた——という体裁。
- だがこの謝罪文は、**監視下に置かれた広報担当者が「書かされている」**もの。担当者の本音の上に、会社の検閲機構が定型謝罪文を貼り重ねて表示している。
- 担当者は検閲の正体（表示層での置換であってデータ消去ではないこと）に気づき、**コピーされたときにだけ原文＝SOSが現れる**よう文面の下に本音を仕込んだ。さらにフィルタの癖（テンプレート一致判定）を学習し、わざとテンプレートに一致しない書き方を混ぜて縫い目を意図的に誘発した。それが**壊れた一文**（後述）。
- 表示は検閲できても、**お問い合わせフォームに送られた文字は「表示」ではなく「記録」として処理される＝検閲の管轄外**。だから真文を貼り付けて送信する＝転載することだけが、SOSを検閲の外へ届ける行動になる。
- ページ冒頭の「本文の無断転載・複製はご遠慮ください」は、最初から**「転載（コピー＆ペースト）こそが脱出行動である」**ことを裏返しに告げていた伏線。これがオチで回収される。
- **過去の犠牲者**: 担当者は初めてではない。前任者も同じ席で本音を書き換えられ、ある日「更新が止まった」（＝消された）。更新履歴の真文にその痕跡が残る。
- **脅している側**: 社長メッセージの真文は、担当者を追い込む監視者（社長）の生の声。「笑え」「一字残らず直させる」。
- **GOOD END**: 真文コア区間を送信すると、記録が検閲を貫通し、照合エラーで検閲が崩壊。ページ全文が真実に差し替わり、真相エピローグ（担当者のその後）が1画面出る。数日後ページは静かに閉じられ、最後の更新履歴に担当者本人の手で一行だけ残る——「原文ママ。」。プレイヤーは読者ではなく、SOSを外へ運び出した**運び屋**だった。

### 1.3 例外は2つで固定（無自覚な例外を作らない）

1. **壊れた一文** … 置換の縫い損ない。前半テンプレ敬語＋後半に原文断片。担当者が意図的に誘発した綻び＝最初に見せる違和感。
2. **社長写真キャプション（画像焼き込み文字）** … 画像の文字はコピーできない＝検閲が完璧に支配する場所。コピーしても何も起きないことが逆に法則を教える。対抗反応の最終段階（区画の画像化）は、この「検閲が完璧な場所」を世界の側が後追いで作る反撃として地続き。

## 3. 判定仕様（決定論的）

- **GOOD条件**: 送信時、textarea の入力内容を正規化した文字列が、正規化した**判定コア区間**を部分文字列として含むとき GOOD。それ以外は BAD。
- **判定コア区間（CORE、確定文字列）**:
  ```
  この文章を、そのまま、下のお問い合わせ欄に貼って送信してほしい。
  ```
- **正規化（determinsitic）**:
  1. 入力・COREの双方から、すべての空白類（半角/全角スペース・タブ・改行 `\s` 全般）を除去する。
  2. 句読点・記号はそのまま残す（除去しない）。
  3. 大文字小文字・全半角の変換は行わない（対象がすべて日本語のため不要）。
  - 判定式: `normalize(textarea).includes(normalize(CORE))`
- **許容範囲**: 前後に余分な文字（真文の他の行・空行・プレイヤーが足したメモ等）が貼り込まれていても、CORE が連続して含まれていればGOOD（部分一致許容）。COREの内部に文字が割り込んだ場合はBAD（連続一致必須）。
- **判定の純度（G-003）**: placeholder・自動補完・プログラム由来文字列は入力値に含めない（`value` のみを見る）。コピー20文字未満で偽文のまま貼った内容や、無関係文はCOREを含まないため自然にBADへ落ちる。
- **copyMin との整合**: apology-body の真文は copyMin（=20）以上のまとまりでコピーして初めて注入される。CORE を含む真文は本文段落を丸ごと写せば必ず取得できる長さ。

## 4. ヒント仕様（4段階・段階的後出し・G-002）

初期状態は操作指示ゼロ。詰まり検知（無操作時間・BAD送信回数）で段階解放する。閾値はURLクエリで上書き可（§6）。

| 段階 | 発火条件 | 演出・確定文言 |
|---|---|---|
| 1. 近接反応（常時・無言） | テキストの長押し選択が始まる | 壊れた一文（apology-body 第4段落）がごくわずかに**にじむ/震える**。文言追加なし。可読性は壊さない（G-001）。 |
| 2. 体感ヒント | 無操作 約45秒 **or** BAD 1回 | 更新履歴に1行追加: 「2026年6月20日　本文中の表現につき『誤植では』とのご指摘を複数いただいておりますが、当該箇所は**原文ママ**です。」 |
| 3. 明示ヒント | 無操作 約90秒 **or** BAD 2回 | 冒頭注意書きが強調表示に変化: 「本ページの文章の無断転載・複製は**固く**ご遠慮ください。」（禁止の強調＝逆説の誘導。操作手順は書かない） |
| 4. 最終救済 | BAD 3回以上 | 壊れた一文の近くに小さな「**引用する**」リンクを出現。タップで当該段落を選択状態にする（＝コピー直前まで運ぶ）。**コピーと貼り付けはプレイヤーに残す**。 |

- 失敗フィードバックは常時可（BAD定型文）。コピー20文字未満で何も起きないこと自体が情報。
- **代替（転写モード）は失敗検知後のみ案内**（クリップボード不成立をコピー無反応/送信失敗で検知した場合に限り、「引用する」経由でtextareaへ直接転記するin-page動作を案内。先出し禁止）。

## 6. パラメータ表（URLクエリ上書き・G-005）

`?copyMin=20&hint1=45&hint2=90&guardStage1=3&guardStage2=6&overwriteGraceMs=4000&debug=1`

| クエリ名 | 意味 | 暫定値 | 備考 |
|---|---|---|---|
| `copyMin` | 真文へ置換する最小コピー文字数 | 20 | 未満は表示どおりの偽文を返す。世界内の意味＝検閲AIの照合限界（短文は即照合できる） |
| `hint1` | ヒント段階2（更新履歴追加）の無操作秒数 | 45 | BAD 1回でも発火 |
| `hint2` | ヒント段階3（注意書き強調）の無操作秒数 | 90 | BAD 2回でも発火 |
| `guardStage1` | 対抗反応 第1段階（更新履歴に「不正な複製を検知しました」＋注意書き硬化）が発火するコピー回数 | 3 | 累計コピー回数で数える |
| `guardStage2` | 対抗反応 最終段階（コピー済み区画の画像化。コア区画は除外）が発火するコピー回数 | 6 | apology-body は対象外 |
| `overwriteGraceMs` | 塗り潰し競争の猶予（ms） | 4000 | **オプション仕様が採用された場合のみ有効**（§7-C） |
| `debug` | デバッグ表示（実測HUD等） | 0（本番）/1 | 実機チューニング後に確定値をコードへ反映し、デバッグ経路は残す |

実機（Xアプリ内ブラウザ）チューニング後に確定値を反映。静的配信のため改修時はキャッシュバスター `?v=` をセット更新（G-011）。

## 7. 画面構成・演出仕様

物理的には **1ページ（縦スクロール）＋送信結果オーバーレイ** の2画面（S1/S2）。進行必須UIはすべて通常フローのスクロールコンテンツ（下部固定UIなし）。可視高は visualViewport 追従（生の100vh不使用・G-006）。

### S1: 謝罪文ページ（メイン・唯一の常設画面）

上から順に、確定テキストを流し込んだ状態:

1. **ヘッダ**: テキストロゴ「株式会社コハク製菓」＋パンくず「HOME ＞ お知らせ ＞ お詫び」。画像不要。
2. **注意書き（notice）**: 「本ページの文章の無断転載・複製はご遠慮ください。」（装飾なし・最重要伏線。段階3で「固く」強調版へ）。
3. **謝罪文本文（apology-body）**: §2.3の偽文。第4段落＝壊れた一文（にじみ/震えは段階1で長押し時のみ・G-001）。**このページの生命線＝コピーでSOS真文＋CORE区間が出る唯一の判定区画**。
4. **社長メッセージ（ceo-msg）**: 顔写真1枚（生成画像）＋キャプションは画像焼き込み（ceo-caption「いつも、笑顔で。代表取締役社長」＝例外2）＋短いコメント（偽文）。
   - **オプション**: 社長叱責の短尺動画（8秒・AI生成、muted autoplay・`poster`に顔写真）。進行必須情報は動画・音声に載せない（G-014）。不採用でも成立を正とする。
5. **FAQ（faq-1〜3）**: §2.4。含みを分散配置（各偽文は1文以内の空虚さで、真相を説明しない）。
6. **更新履歴（history）**: §2.6。初期2行＋段階2で「原文ママ」行＋対抗反応で「不正な複製を検知」行。
7. **お問い合わせセクション（contact）**:
   - 見出し「書面でのお問い合わせをご希望のお客様は、以下の返送先住所をコピーの上、ご利用ください。」＋住所（contact-address）。**コピー操作の自然な練習導線＝最初の発覚点**（貼ると案B真文に化ける）。チュートリアルとは名乗らない。
   - 自由記入 textarea（placeholder「お問い合わせ内容をご記入ください」）＋「送信する」ボタン（進行必須UI・スクロールコンテンツ）。

### S2: 送信結果オーバーレイ（可変高＋`overflow-y:auto`前提・G-004）

- **BAD**: §2.8の定型受領文のみの素っ気ないモーダル。「戻る」でS1へ。ヒントは書かない。（オプションで社長叱責動画・ミュート8秒を差す案もあるが、不採用でも成立する設計を正とする。）
- **GOOD**: §2.9の(A)検閲崩壊演出 → S1全文が真文に差し替わって再表示 → (B)真相エピローグ1画面 →(C)回収の一文。**最大の見せ場をGOODに置く（G-007）**。
  - **全文真実化時の notice 差し替え（回収文・確定文言）**: S1 再表示の際、冒頭注意書き（notice）は「――写した文字だけが、ほんとうのことを言う。」へ静かに差し替える。これは演出上の回収文であり、notice の「真文」ではない（§2.2・§5 の「notice=真文なし」と矛盾しない）。
  - **全文真実化時の更新履歴**: 更新履歴（history）は日付行を消し、§2.6 の真文（過去の犠牲者の痕跡）1件のみの表示に差し替える。

### 対抗反応の段階演出（密輸の手触り。コピー累計回数で発火）

| 段階 | 閾値 | 演出 |
|---|---|---|
| 1 | `guardStage1`（=3） | 更新履歴に「不正な複製を検知しました」行が増える／注意書きが硬化（確定文言は §2.2 硬化版「本ページの文章の無断転載・複製を**固く禁じます**。複製行為はすべて記録されます。」。段階3強調と地続きで、発火時は段階3文言を上書きする） |
| 2（最終） | `guardStage2`（=6） | コピー**済み**区画のテキストを画像に差し替えコピー不能化（例外2「検閲が完璧な場所」を世界が後追いで作る反撃）。**コア区画 apology-body は絶対に画像化しない（詰み防止）** |

### 塗り潰し競争（オプション仕様。実装優先度・低）

- 真文をtextareaに貼ると検閲が気づき、貼った文字が端から定型文へ塗り替えられ始める。塗り切られる前に送信、という最後の緊張演出。猶予は `overwriteGraceMs`（=4000）程度のゆるい設計で、反射神経は要求しない。失敗してもペナルティなし・即再試行可。
- 端末差・パフォーマンス差による理不尽化リスク（G-006系）があるため**オプション仕様**とし、**実機＋ブラインドプレイテストで採否判断**。不採用でも本編は完全に成立する（`overwriteGraceMs` は採用時のみ有効）。
