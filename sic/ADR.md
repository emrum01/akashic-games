# 『原文ママ』（sic）ADR — 設計判断記録

軽量ADR形式（日付／ステータス／文脈／決定／結果・注意）。体験仕様の「現在形の正」は `DESIGN.md`、企画の正は `企画.md`。本書は「なぜ今そうなっているか」の経緯を残す（G-013）。

初稿は `企画.md` の「初期ADR候補」15項目を移送。実装中に下した追加判断は ADR-16 以降に追記する。

## ADR-01: コア動詞は「書き写す（コピー＆ペースト）」

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: 過去作と動詞・違和感型・デバイス機能を被らせない新作が必要。X経由で遊ばれる前提。
- 決定: コア動詞を「書き写す（コピーして貼る）」とし、センサー・マイク・音を進行条件にしない（G-014）。
- 結果・注意: Xの「引用して晒す」文化と一致しチュートリアルコスト極小。使用機能はクリップボード（copy/paste）のみ。

## ADR-02: 検閲＝本音を定型テンプレートへ置換する表示層の処理

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: 世界律を1文に収束させ、全要素をそこから導出する（恣意的な足し算を禁じる）。
- 決定: 「検閲は表示層で本音を定型テンプレへ置換する処理であり、文字データそのものは消していない」を技術的根拠とする。導出できない演出・設定は足さず削る。
- 結果・注意: 「コピー（データ層に触れる唯一の行為）だけが原文を運び出せる」という必然が全ギミックの土台になる。

## ADR-03: コピー発動閾値は「一度に20文字以上」

- 日付: 2026-07-13
- ステータス: Accepted（暫定値・実機で確定）
- 文脈: 単語コピー等の通常利用で異変が起きると導線が壊れる。閾値は机上で確定できない。
- 決定: 選択・コピー文字数が `copyMin`（=20）以上のとき真文へ置換。未満は表示どおり。世界内の意味＝検閲AIの照合限界。
- 結果・注意: URLクエリ `copyMin` で上書き可（G-005）。住所（約42字）で初めて発覚する導線が立つ。実機チューニング後に確定値を反映。

## ADR-04: 判定は textarea への貼り付け内容のみ

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: プレースホルダ・自動補完等のプログラム由来文字列を判定に混ぜると純度が壊れる（G-003）。
- 決定: 送信時、`textarea.value` を正規化した文字列が正規化 CORE を部分文字列として含めば GOOD。それ以外は BAD。正規化は「空白類 `\s` 全除去・句読点は残す」（DESIGN §3）。
- 結果・注意: 前後の余分な貼り込みは許容、CORE 内部への割り込みは BAD。placeholder は `value` に含まれないため自然に除外される。

## ADR-05: ヒントは4段階の後出し

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: 操作指示・答えを初期UIに書くと謎が謎でなくなる（G-002）。
- 決定: (1) 近接反応（常時・無言・壊れた一文のにじみ）→(2) 体感（無操作45s/BAD1回で更新履歴に「原文ママ」行）→(3) 明示（無操作90s/BAD2回で注意書き「固く」強調）→(4) 最終救済（BAD3回で「引用する」リンク）。
- 結果・注意: 閾値は `hint1`/`hint2` で上書き可。操作手順そのものは最後まで書かない。

## ADR-06: 代替の転写モードは失敗検知後にのみ案内

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: WebView等でクリップボードが機能しない環境がありうる。先出しすると謎が台無し。
- 決定: クリップボード不成立（copy無反応/`clipboardData` 欠如/BADで偽本文がそのまま貼られた）を検知した場合に限り、「引用する」経由で textarea へ直接転記する in-page 動作を案内。
- 結果・注意: 先出し禁止。デスクトップは Ctrl/Cmd+C で同一 copy イベントが発火するため追加対応不要。

## ADR-07: GOOD END に最大の見せ場（非対称を GOOD 側に倒す）

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: BAD を作り込むほど GOOD が地味になり「正解ルートが損」に見える（G-007）。
- 決定: GOOD＝検閲崩壊演出→全文真実化→真相エピローグ→回収の一文。BAD は素っ気ない定型受領文のみ・ヒントなし。
- 結果・注意: 両END とも可変高＋`overflow-y:auto`（G-004）。

## ADR-08: 例外は2つで固定

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: 無自覚な例外を作ると世界律が濁る。
- 決定: 例外は (1) 壊れた一文（置換の縫い損ない）と (2) 画像文字（社長写真キャプション＝検閲が完璧な場所）の2つのみ。
- 結果・注意: 対抗反応の最終段階（区画の画像化）は例外2を世界が後追いで作る反撃として地続きに設計する（矛盾させない）。

## ADR-09: 真文・オチの結論は DESIGN.md に記録

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: 世界律・オチは制作中に反転しうる。本文に結論を書き込むと反転時に全文書き直し（G-008）。
- 決定: 核・真相の全文は DESIGN.md にのみ書き、ゲーム内テキスト（偽文）は各画面1文以内の含みに留める。
- 結果・注意: プレイヤーはクリア後に核を言い当てられなくてよい（体感で一貫していればよい）。

## ADR-11: 壊れた一文＝置換の縫い損ない

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: 「誤植」に見える一文を、書き手の雑さでなく検閲の機構的痕跡として導出する。
- 決定: 前半は完全な定型敬語、`——` を継ぎ目に後半だけ常体・二人称の生の問いへ崩れる。担当者がフィルタの癖を突いて意図的に誘発した綻びという設定で統一。
- 結果・注意: 敬語崩壊はこの1文に閉じ、前後の可読性は完全維持（G-001）。後半断片は真相を明かさず匂わせるに留める。

## ADR-12: 真文は区画別に虫食い配置（探索化）

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: 単発仕掛けでなくページ全体を虫食いモザイクにして「懐中電灯を当てて回る探索」にする。
- 決定: 謝罪文本文=SOS（判定対象）／FAQ=担当者の状況／社長メッセージ=脅す側の声／更新履歴=犠牲者の痕跡／返送先住所=最初の発覚点。
- 結果・注意: 全区画の閲覧・コピーはクリア必須にしない。判定対象は apology-body の CORE 区間のみ。

## ADR-13: コピー回数に応じた段階的対抗反応

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: 密輸の背徳感を演出。ただし詰みを絶対に作らない。

## ADR-15: 社長叱責動画（AI生成）はオプション演出

- 日付: 2026-07-13

## ADR-16: 社長写真は CSS プレースホルダ＋キャプションは選択不能テキストで代替

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: 生成画像1枚は後工程。だが例外2（画像焼き込みキャプション＝コピー不能）は初稿から体現させたい。
- 決定: 顔写真は CSS グラデーション＋擬似シルエットで表現。キャプション「いつも、笑顔で。代表取締役社長」は `user-select:none` / `-webkit-user-select:none` / `-webkit-touch-callout:none` / `pointer-events:none` の DOM テキストで、画像焼き込み相当の選択不能性を再現。
- 結果・注意: コピー対象区画リストにも含めない＝真文を持たない（例外2）。生成画像への差し替え時はキャプションを焼き込み、この DOM 代替を撤去する。

## ADR-17: 複数区画にまたがる選択は「区画ごとに真文へ置換」して連結

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: DESIGN では複数区画にまたがる選択の挙動は実装時決定とされた。
- 決定: copy 時、`Selection.containsNode(el, true)`（部分含有可）で選択と交差する区画を DOM 出現順に集め、真文を持つ区画はその真文全文へ、持たない区画は表示文へ置換し、`\n\n` で連結してクリップボードへ書き込む。真文を持つ区画に一つも触れていなければ既定動作（表示どおり）。
- 結果・注意: 単一区画選択が大半の実運用では「その区画の真文全文」がそのまま入る。判定は CORE の部分一致なので、複数区画連結でも apology-body の CORE が含まれれば GOOD。区画境界に依存しない堅牢な判定。

## ADR-18: 対抗反応の画像化は canvas 描画で実装

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: 「テキストを画像に差し替えコピー不能化」を CSS/canvas どちらでも可とされた。
- 決定: canvas で要素の computed font・幅に合わせてテキストを行折り返し描画し、`toDataURL` の `<img>` へ差し替え、`user-select:none` を付与。GOOD END の全文真実化時は `imgized` を解除して真文へ差し替える。
- 結果・注意: 対象は FAQ・社長メッセージ・更新履歴・返送先住所のみ（コピー済みのもの）。apology-body は IMAGIZABLE から除外し詰みを防止。devicePixelRatio は最大2に制限（巨大 dataURL 回避）。ブロック子要素（li 等）は行単位で computed color を保持して描画する（更新履歴 `.sysnote` の赤色強調を画像化後も維持。レビュー修正 2026-07-13）。

## ADR-19: 発火順序 — 「引用する」リンクは通常=段落選択／転写モード=直接転記

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: ヒント段階4（BAD3回）と代替転写モード（クリップボード不成立検知）は、どちらも「引用する」リンクを出すが役割が異なる。
- 決定: 単一のリンク要素を状態フラグ `transcribeMode` で分岐。通常はタップで apology-body 段落を選択状態にする（コピー/貼り付けはプレイヤーに残す）。転写モードでは textarea へ真文を直接転記し、ラベルを「引用する（うまくコピーできない場合）」へ変える。
- 結果・注意: 先出し禁止は維持（BAD3回 or 失敗検知まで非表示）。

## ADR-20: 静的単体HTML・キャッシュバスターは配信時に付与

- 文脈: 単体HTML1ファイルで外部依存なし（file://でも動作）。近隣ゲーム同様の静的配信慣例（G-011）。

## ADR-21: 転写モードの失敗検知は「真文注入試行後」に限定（レビュー修正）

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: 独立レビューで、偽文を手打ち送信しただけで偽本文断片の部分一致により「クリップボード注入失敗」と誤検知し、「引用する」リンクが早期露出する不具合が指摘された（G-002 先出し禁止に抵触）。
- 決定: `truthCopyAttempted` フラグを導入し、copy イベントで真文注入を実際に試行した後に限り、BAD送信内容の偽本文断片一致を「注入不成立」と判断して転写モードを案内する。
- 結果・注意: 偽文の手打ち・目視転記ではリンクは出ない。ADR-06 の「失敗検知後にのみ案内」の検知条件を厳密化したもの。

## ADR-23: 状態は非永続・単一セッション完結

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: リロードでコピー回数・BAD回数・ヒント段階・対抗反応がすべてリセットされる（localStorage 等の永続化なし）。
- 決定: 3〜5分の単発体験として意図的に単一セッション完結とする。永続化は詰み（画像化状態の持ち越し等）や検証コストを増やすだけで益がない。
- 結果・注意: リロードは事実上の「最初からやり直す」操作を兼ねる。対抗反応の最終段階（画像化）もリロードで解除される＝恒久的詰みが構造的に発生しない。

## ADR-24: クリップボード無音失敗の検知拡張と、実キーストローク検証の結論

- 日付: 2026-07-13
- ステータス: Accepted
- 文脈: ブラインドプレイテスト（Playwright実操作）で「選択→Ctrl+C→Ctrl+V で textarea が空のまま」事象が報告された。切り分けの結果、**テスト環境の制約（操作誤り相当）**と結論: macOS の headless Chromium では `Control+c` はコピーのアクセラレータではなく copy イベント自体が発火しない（copyCount=0・注入未到達）。正しいアクセラレータ `Meta+c`（Playwrightでは `ControlOrMeta+c`）を使えば、実キーストローク経路は clipboard-read/write 権限の有無に関わらず end-to-end で成立する（copy発火→setData注入→Ctrl(Meta)+V で真文277字が貼付されることを実測）。実装バグではない。
- 決定: ただし「copy は成立して見えるがクリップボードに書き込まれない端末」（WebView等の無音失敗）は現実に存在しうるため、転写モード検知を拡張する。真文注入試行（`truthCopyAttempted`）後に限り、(a) 偽本文断片がそのまま貼られた、(b) textarea への paste イベントが空文字だった、(c) 実質空のまま送信された——のいずれかで `clipboardBroken` とし転写モードを案内する。
- 結果・注意: 発動は従来どおり失敗検知後のみ・先出し禁止（G-002）。`clipboardBroken=true` 時は「引用する」リンク（ヒント段階4と同一要素）が転写動作（textareaへ直接転記）になるため、クリップボード全滅端末でもクリア経路が閉じない（詰み防止の恒久保証）。paste 検知は textarea の paste イベントの clipboardData のみを見る（判定純度 G-003 は送信時 `value` 参照のままで不変）。
